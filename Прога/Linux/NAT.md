Network Address Translation (NAT) — это механизм в сетевых системах, предназначенный для преобразования IP-адресов и, при необходимости, портов пакетов, проходящих через сетевой узел. В Linux NAT реализован в рамках подсистемы Netfilter и тесно взаимодействует с conntrack, что позволяет отслеживать состояния соединений и корректно перенаправлять трафик. Основная цель NAT — обеспечивать возможность взаимодействия локальных сетей с внешними сетями с разными адресными пространствами, скрывая внутренние адреса и обеспечивая масштабирование сетевой инфраструктуры. Помимо маскарадинга (массовое преобразование исходных адресов локальной сети в один внешний), NAT используется для перенаправления портов, когда пакеты, направленные на внешний адрес и порт, перенаправляются на другой внутренний адрес и порт. Механизм позволяет сохранять согласованность соединений: ответные пакеты корректно возвращаются к инициатору, а изменение адресов и портов прозрачно для приложений.

NAT в Linux делится на несколько видов. Source NAT (SNAT) изменяет исходный адрес пакета, обычно используемый при выходе из локальной сети в интернет. Destination NAT (DNAT) изменяет целевой адрес и/или порт пакета, применяемый для перенаправления входящих соединений на внутренние серверы. Masquerade — частный случай SNAT, когда адрес источника подставляется автоматически, часто используется для динамических внешних интерфейсов. Все эти виды NAT интегрируются с таблицей nat Netfilter и взаимодействуют с цепочками PREROUTING, POSTROUTING и OUTPUT, что позволяет изменять пакеты как на входе, так и на выходе. При этом conntrack отслеживает связь между оригинальными и изменёнными адресами, обеспечивая корректную маршрутизацию ответного трафика.

Структурно работа NAT выглядит следующим образом: пакет входит на интерфейс, попадает на соответствующий hook Netfilter (PREROUTING для DNAT, POSTROUTING для SNAT), где проверяется таблица NAT. Если правила совпадают, адреса или порты изменяются, а запись о трансляции заносится в таблицу соединений conntrack. Эта запись хранит исходный и преобразованный адреса, порты, протокол и состояние соединения. Когда пакет возвращается от внешнего узла, conntrack сопоставляет его с существующей записью и восстанавливает оригинальный адрес и порт, чтобы пакет был корректно доставлен внутреннему приложению.

Схематично:
Внутренняя сеть        NAT/Conntrack       Внешняя сеть
192.168.1.10:5000 ──►  SNAT ──► 203.0.113.5:6000
192.168.1.11:5001 ──►  Masquerade ──► 203.0.113.5:6001
Входящие пакеты ◄── DNAT/Conntrack ◄── Внешняя сеть:203.0.113.5:6000

**SNAT (Source NAT)** — преобразование исходного адреса пакета для выхода из локальной сети. Позволяет нескольким внутренним хостам использовать один внешний IP.

**DNAT (Destination NAT)** — преобразование целевого адреса и/или порта пакета. Используется для перенаправления входящего трафика на внутренние серверы.

**Masquerade** — разновидность SNAT для динамических внешних адресов, автоматически подставляющая исходный адрес интерфейса.

**Таблица NAT** — логическая структура в Netfilter для хранения правил трансляции адресов. Разделена по цепочкам PREROUTING, POSTROUTING и OUTPUT.

**Port forwarding (перенаправление портов)** — процесс изменения целевого порта пакета для маршрутизации на внутренний сервер. Обеспечивает доступ из внешней сети к внутренним сервисам без изменения внешнего адреса.

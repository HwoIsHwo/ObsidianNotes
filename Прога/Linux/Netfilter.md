Netfilter - это подсистема ядра Linux, предназначенная для реализации фильтрации пакетов, сетевого адресного преобразования (NAT) и ведения состояния соединений (connection tracking). Она функционирует как промежуточный слой между сетевым стеком ядра и сетевыми интерфейсами, перехватывая проходящие через систему пакеты и позволяя модифицировать их, блокировать или перенаправлять в зависимости от заданных правил. 
Основное назначение Netfilter — управление сетевым трафиком на уровне ядра. С точки зрения архитектуры, Netfilter строится вокруг механизма «хуков» (hooks), которые размещены в ключевых точках обработки IPv4, IPv6 и ARP-пакетов: пакеты могут быть перехвачены при входе на интерфейс (PREROUTING), перед отправкой из локального процесса (OUTPUT), при маршрутизации (FORWARD), после обработки локальными процессами (POSTROUTING).

Функционально Netfilter реализует три основные возможности: фильтрацию пакетов по заданным правилам (iptables/ nftables), трансляцию адресов и портов (NAT) и ведение состояния соединений (conntrack). Фильтрация позволяет блокировать или разрешать трафик на основе IP-адреса, порта, протокола или более сложных критериев, включая состояния соединения. NAT используется для изменения исходного или целевого адреса пакета, что актуально для маршрутизации внутренних сетей в интернет. Conntrack отслеживает состояния TCP/UDP соединений, позволяя применять динамические правила фильтрации, например, разрешать только ответные пакеты от уже установленных соединений. Встроенные в Netfilter таблицы и цепочки обеспечивают структурированное хранение правил, что упрощает их управление и позволяет комбинировать статические и динамические фильтры.

Структура Netfilter включает несколько ключевых компонентов: хуки ядра, таблицы правил, цепочки (chains) и обработчики состояний (connection tracking). Хуки — это точки интеграции в сетевой стек, таблицы объединяют правила по назначению (например, filter для фильтрации, nat для NAT, mangle для модификации заголовков), цепочки определяют порядок применения правил в рамках таблицы, а conntrack регистрирует состояния соединений и позволяет применять условия фильтрации. Система правил работает последовательно: каждый пакет проверяется по цепочкам таблицы, и действие (ACCEPT, DROP, REDIRECT, MASQUERADE) применяется при первом совпадении с правилом.

Схематично это можно представить следующим образом:
Сетевой интерфейс
       │
       ▼
    PREROUTING ──> NAT/Filter
       │
       ▼
   FORWARD ──> Filter
       │
       ▼
    INPUT ──> Filter
       │
       ▼
   Локальные процессы
       │
       ▼
    OUTPUT ──> Filter/NAT
       │
       ▼
    POSTROUTING ──> NAT/Filter
       │
       ▼
    Сетевой интерфейс

Хук (Hook) — точка интеграции в сетевой стек ядра, позволяющая перехватывать пакеты на разных стадиях их обработки. Используется для применения правил Netfilter до или после маршрутизации.

Цепочка (Chain) — упорядоченный список правил, через который последовательно проходит пакет. Каждая цепочка привязана к конкретному хуку и выполняет проверку по заданным условиям.

Таблица (Table) — логическая группа цепочек, объединенная по назначению, например для фильтрации, NAT или модификации пакетов. Таблицы структурируют правила и определяют их область применения.

Conntrack (Connection Tracking) — механизм отслеживания состояний сетевых соединений. Позволяет применять правила фильтрации на основе состояния соединения, например, разрешать только ответные пакеты.

NAT (Network Address Translation) — процесс изменения исходного или целевого адреса и/или порта пакета. Используется для маршрутизации пакетов между сетями с различными адресными пространствами.

Iptables — это утилита и фреймворк для конфигурирования и управления подсистемой Netfilter в Linux. Она позволяет задавать правила фильтрации пакетов, трансляции адресов и перенаправления трафика, используя цепочки и таблицы ядра. Iptables работает как интерфейс между пользователем и ядром, предоставляя удобный способ создавать, модифицировать и удалять правила, которые обрабатываются на уровне Netfilter hooks. Основная архитектура iptables строится вокруг таблиц, каждая из которых содержит цепочки (chains), в которых хранятся правила, определяющие действия для пакетов. Действия, применяемые к пакетам, называются целями (targets), среди которых есть ACCEPT, DROP, RETURN, а также специальные цели для NAT и перенаправления, включая REDIRECT.

Цель REDIRECT используется в таблице `nat` для перенаправления пакетов на другой порт локального хоста. В отличие от DNAT, который изменяет целевой адрес и/или порт пакета, REDIRECT всегда меняет только порт, оставляя адрес локального интерфейса. Это удобно для реализации локальных прокси-серверов или перенаправления трафика на внутренние сервисы, работающие на нестандартных портах. Реализация REDIRECT в ядре Linux основана на hook PREROUTING для входящих пакетов и OUTPUT для локально генерируемых пакетов. При совпадении пакета с правилом REDIRECT ядро изменяет поле TCP/UDP порта в заголовке пакета и добавляет запись в таблицу conntrack, чтобы сохранить состояние соединения и обеспечить корректную обработку ответных пакетов.

С точки зрения структуры ядра, REDIRECT реализован как встроенный target модуля `iptables` в пространстве ядра. Когда пакет проходит через цепочку NAT, механизм REDIRECT изменяет поле `dest port` в структуре `sk_buff` пакета. Взаимодействие с conntrack гарантирует, что для каждого перенаправленного соединения создается соответствующая запись состояния, которая хранит оригинальный и изменённый порт. В результате пакеты, возвращаемые от локального сервиса, автоматически маршрутизируются обратно к инициатору, без необходимости дополнительной настройки. REDIRECT также учитывает адрес интерфейса, на который пришел пакет, обеспечивая локальную доставку на том же узле, что и входящий трафик.

Схематично:
Внешний пакет ──► PREROUTING ──► REDIRECT (dst port 5000 -> 6000) ──► Локальный сервис:6000
  |                                                                      
  └─> Conntrack сохраняет связь: (orig port 5000 ↔ redirected port 6000)

**Iptables** — утилита для конфигурации подсистемы Netfilter, позволяющая создавать правила фильтрации, NAT и перенаправления пакетов. Предоставляет интерфейс для управления таблицами и цепочками ядра.

**Цепочка (Chain)** — упорядоченный набор правил, через который последовательно проходят пакеты. Каждая цепочка привязана к определённому хуку ядра и определяет порядок применения правил.

**Target (цель)** — действие, выполняемое над пакетом при совпадении с правилом. Включает стандартные действия (ACCEPT, DROP) и специальные, такие как REDIRECT и DNAT.

**REDIRECT** — специальная цель iptables, изменяющая порт пакета на локальном хосте. Используется для перенаправления трафика на локальные сервисы без изменения адреса.

**PREROUTING** — hook Netfilter, через который проходят входящие пакеты до маршрутизации. Используется для применения NAT и целей типа REDIRECT на входящем трафике.

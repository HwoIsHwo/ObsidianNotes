Динамический массив с непрерывным хранением. Основной рабочий контейнер STL. Быстрый доступ по индексу, хорош для численных данных и плотных структур.

Вектор представляет контейнер, который содержит коллекцию объектов одного типа. Для работы с векторами необходимо включить заголовок:
```
#include <vector>
```

В угловых скобках указывается тип, объекты которого будут храниться в векторе. То есть вектор numbers хранит объекты типа int. Однако такой вектор пуст. Он не содержит никаких элементов.

Но мы можем инициализировать вектор одним из следующих способов:
```
std::vector<int> v1;              // пустой вектор

std::vector<int> v2(v1);          // вектор v2 - копия вектора v1

std::vector<int> v3 = v1;         // вектор v3 - копия вектора v1

std::vector<int> v4(5);           // вектор v4 состоит из 5 чисел, каждое число равно 0

std::vector<int> v5(5, 2);        // вектор v5 состоит из 5 чисел, каждое число равно 2

std::vector<int> v6{1, 2, 4, 5};  // вектор v6 состоит из чисел 1, 2, 4, 5

std::vector<int> v7 = {1, 2, 3, 5}; // вектор v7 состоит из чисел 1, 2, 3, 5
```

Важно понимать отличие в данном случае круглых скобок от фигурных:
```
std::vector<int> v1(5);       // вектор состоит из 5 чисел, каждое число в векторе равно 0

std::vector<int> v2{5};       // вектор состоит из одного числа, которое равно 5

std::vector<int> v3(5, 2);    // вектор состоит из 5 чисел, каждое число равно 2

std::vector<int> v4{5, 2};    // вектор состоит из двух чисел 5 и 2
```

При этом можно хранить в векторе элементы только одного типа, который указан в угловых скобках. Значения других типов в вектор сохранить нельзя.

### Обращение к элементам и их перебор
Для обращения к элементам вектора можно использовать разные способы:
- \[index]: получение элемента по индексу (также как и в массивах), индексация начинается с нуля
- at(index): функция возращает элемент по индексу
- front(): возвращает первый элемент
- back(): возвращает последний элемент

При этом следует учитывать, что индексация не добавляет элементов. Например, если вектор содержит 5 элементов, то мы не можем обратиться к шестому элементу. При таком обращении результат неопределен. Некоторые комиляторы могут генерировать ошибку, некоторые продолжат работать, но даже в этом случае такое обращение будет ошибочно, и оно в любом случае не добавит в вектор шестой элемент.
Чтобы избежать подобных ситуаций, можно использовать функцию at(), которая хотя также возвращает элемент по индексу, но при попытке обращения по недопустимому индексу будет генерировать исключение out_of_range:
```
#include <iostream>
#include <vector>
#include <stdexcept>

int main(){
    std::vector<int> numbers { 1, 2, 3, 4, 5};
    
    try{
        int n = numbers.at(8);
    }catch (std::out_of_range e){
        std::cout << "Incorrect index" << std::endl;
    }
}
```


Доступ к элементам
```
v[i];        // без проверок
v.at(i);     // с проверкой, бросает std::out_of_range
v.front();   // первый элемент
v.back();    // последний элемент
v.data();    // T*
```


Размер и ёмкость
```
v.size();      // количество элементов
v.capacity();  // размер буфера
v.empty();     // size == 0
v.max_size();  // теоретический предел
```


Управление памятью
```
v.reserve(100);
```
Выделяет память под 100 элементов, **не создавая их**.
```
v.shrink_to_fit();
```
Запрос на освобождение лишней ёмкости. Нестрогий: может ничего не сделать.


Добавление элементов
```
v.push_back(10);
v.emplace_back(3, 'x');
```
1 - Добавляет копированием или перемещением.
2 - Конструирует элемент **на месте**, без временного объекта.


Удаление элементов
```
v.pop_back();
v.clear();
```
1 - Удаляет последний элемент. Размер уменьшается, ёмкость — нет.
2 - Удаляет все элементы, но память сохраняется.


Вставка и удаление в середине
```
v.insert(pos, value);
v.insert(pos, count, value);
v.insert(pos, first, last);

std::vector<int> v = {1,2,4};
auto it = v.begin() + 2;
v.insert(it, 3); // {1,2,3,4}

v.erase(pos);
v.erase(first, last);
std::vector<int> v = {1,2,3,4,5};
v.erase(v.begin() + 1); // {1,3,4,5}
```


Итераторы
```
v.begin(), v.end();
v.cbegin(), v.cend();
v.rbegin(), v.rend();

for (auto it = v.begin(); it != v.end(); ++it) {
    std::cout << *it;
}
```

### Инвалидация
Критичный момент, часто источник ошибок.
- `push_back`, `emplace_back`, `insert` → могут инвалидировать **все** итераторы и ссылки.
- `erase` → инвалидирует итераторы на удалённые элементы и дальше.
- `reserve` → инвалидирует всё.
- `clear` → элементы уничтожены, но `data()` остаётся валидным до следующей модификации.

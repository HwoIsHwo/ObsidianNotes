`std::unordered_map<Key, T>` — ассоциативный контейнер на основе **хеш-таблицы**.
Хранит пары `key → value` с **уникальными ключами**, порядок элементов не гарантируется.

Средняя сложность:
- поиск — `O(1)`
- вставка — `O(1)`
- удаление — `O(1)`

В худшем случае (плохой хеш) — `O(n)`.

## Когда использовать
Берём `unordered_map`, если:
- порядок не важен
- нужен максимально быстрый поиск
- нет диапазонных запросов
- ключ хорошо хешируется

Если нужен порядок или `lower_bound` — тогда `map`.
# Вставка
### 1. `insert`
`m.insert({"one", 1});`
Если ключ уже существует — вставка не произойдёт.

### 2. `emplace`
`m.emplace("two", 2);`
Конструирует значение на месте.

### 3. `operator[]`
`m["three"] = 3;`
Если ключа нет — создаётся значение по умолчанию.
`int x = m["unknown"]; // создаст {"unknown", 0}`
Это часто неожиданное поведение.

# Доступ
### Через `[]`
`std::cout << m["one"];`
Создаёт элемент при отсутствии ключа.

### Через `at`
`std::cout << m.at("one");`
Бросает исключение, если ключа нет.

### Через `find`
`auto it = m.find("one"); if (it != m.end()) {     std::cout << it->second; }`
Предпочтительный способ для проверки существования.

# Итерация
`for (const auto& [key, value] : m) {     std::cout << key << " -> " << value << "\n"; }`
Порядок произвольный и может меняться после rehash.

# Удаление
`m.erase("one"); m.erase(it); m.clear();`

`std::map<Key, T>` — ассоциативный контейнер, который хранит пары  
`key → value` с **уникальными ключами** и поддерживает их в **отсортированном порядке**.

Реализация почти всегда — **красно-чёрное дерево** (сбалансированное бинарное дерево поиска).
Основные свойства:
- элементы отсортированы по `Compare` (по умолчанию `std::less<Key>`)
- поиск, вставка, удаление — `O(log n)`
- итераторы не инвалидируются при вставке (кроме удаляемого элемента)
- не поддерживает случайный доступ по индексу

`map` нужен когда:
- требуется упорядоченность по ключу
- нужен поиск по диапазону (`lower_bound`, `upper_bound`)
- важна стабильность итераторов
- размер данных умеренный 
Если порядок не важен и нужна скорость — чаще берут `unordered_map` (`O(1)` в среднем).

Методы:
Вставка insert
```C++
#include <map>
#include <string>
#include <iostream>

std::map<std::string, int> m;

m.insert({"one", 1});

auto [it, inserted] = m.insert({"one", 1});
if (!inserted) {
    std::cout << "key already exists\n";
}
```
Возвращает `pair<iterator, bool>` — вставился ли элемент. Если ключ уже есть — ничего не перезаписывается.

Вставка emplace
```
m.emplace("two", 2);
```
Предпочтительнее `insert`, когда создаётся объект.

operator\[]
```
m["three"] = 3;

int x = m["unknown"]; // создаст {"unknown", 0}

std::cout << m["one"] << "\n";
```
Важно: если ключа нет — создаётся элемент со значением по умолчанию.

at
```
std::cout << m.at("one") << "\n";
```
Если ключа нет — бросит `std::out_of_range`.

find
```
auto it = m.find("one");
if (it != m.end()) {
    std::cout << it->second << "\n";
}
```

Итератор
```
for (const auto& [key, value] : m) {
    std::cout << key << " -> " << value << "\n";
}
```
Порядок — отсортированный по ключу.

Удаление
```
m.erase("one");        // по ключу
m.erase(it);           // по итератору
m.clear();             // удалить всё
```

Поиск
```
auto it = m.lower_bound("two");  // >= "two"
auto it2 = m.upper_bound("two"); // > "two"

Диапазон
for (auto it = m.lower_bound("a");
     it != m.upper_bound("z");
     ++it) {
}
```

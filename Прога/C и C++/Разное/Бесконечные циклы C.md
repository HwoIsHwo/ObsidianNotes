В C «бесконечный цикл» — это цикл, у которого условие всегда истинно или вообще отсутствует. Реализаций несколько, но с точки зрения кода и генерации они неравнозначны.


```C
for (;;) {
    // тело
}
```
Классический и самый идиоматичный вариант в C. Все три выражения цикла опущены, условие считается истинным. Компилятор генерирует простой безусловный переход в начало блока. Чаще всего используется в системном коде, ядрах, драйверах.
Плюсы: читаемо для C-разработчика, не зависит от логических констант.  
Минусы: новичкам менее очевидно.


```C
while (1) {
    // тело
}
```
Условие — константа `1`, то есть «истина». После оптимизации это тот же безусловный цикл.
Иногда пишут `while (true)`, но это уже C99+ с `#include <stdbool.h>` или C++.
С точки зрения машинного кода обычно эквивалентно `for(;;)`.


```C
do {
    // тело
} while (1);
```
Тоже бесконечный, но тело гарантированно выполняется хотя бы один раз.  
Используется реже — обычно когда логика цикла естественно «постусловная».


```C
loop:
    // тело
    goto loop;
```
Фактически то же самое — безусловный переход. Используется редко, чаще в низкоуровневом коде или при ручной развёртке состояний.


## Оптимизации компилятора
Если компилятор видит цикл без побочных эффектов:
`for (;;);`
он может:
- удалить его,
- превратить в `ud2`/trap,
- слить с соседним кодом.

Чтобы цикл гарантированно не удалялся (например, в embedded), добавляют:
```
for (;;)
    __asm__ volatile("nop");
```
или обращение к `volatile` объекту.

Лямбда-функции в C++ — это способ объявить анонимный функциональный объект прямо в месте использования. По сути компилятор генерирует небольшой класс-функтор с перегруженным `operator()`. Основная цель — передать поведение как значение, не создавая отдельную функцию или класс.
```
[capture](params) -> return_type {
    body
}


auto f = [](int a, int b) {
    return a + b;
};
std::cout << f(2, 3);
```
Компилятор превращает это примерно в такой объект:

```
struct __lambda {
    int operator()(int a, int b) const {
        return a + b;
    }
};
```

То есть лямбда — это просто объект с функцией вызова.
Самая важная часть — это capture-список. Он определяет, какие переменные из внешней области видимости будут доступны внутри лямбды и как именно.
Пример захвата по значению:
```
int x = 10;

auto f = [x]() {
    std::cout << x << std::endl;
};
```
Внутри объекта лямбды появляется поле-копия `x`. Это значение фиксируется в момент создания лямбды.

Захват по ссылке:
```
int x = 10;

auto f = [&x]() {
    x++;
};
```
Теперь внутри хранится ссылка, и изменения влияют на исходную переменную.

Можно использовать автоматический захват:
```
[=]   // всё по значению
[&]   // всё по ссылке
```

По умолчанию `operator()` у лямбды является `const`. Поэтому изменять захваченные по значению переменные нельзя. Для этого используется `mutable`.
```
int x = 5;

auto f = [x]() mutable {
    x++;
    return x;
};
```
Это изменяет внутреннюю копию, а не оригинальную переменную.

Лямбды могут иметь явный тип возвращаемого значения:
```
auto f = [](int a, int b) -> double {
    return (a + b) / 2.0;
};
```
Обычно компилятор сам выводит тип, но он обязателен, если тело содержит несколько `return` с разными типами.

В многопоточности:
```
std::thread t([]{
    std::cout << "worker\n";
});
```
Лямбда позволяет передать тело потока без отдельной функции.

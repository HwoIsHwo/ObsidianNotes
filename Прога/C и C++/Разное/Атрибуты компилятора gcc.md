Атрибуты в GNU Compiler Collection позволяют управлять генерацией кода, размещением объектов, проверками и соглашениями о вызовах на уровне конкретной функции, переменной или типа. Синтаксис классический — `__attribute__((...))`, в C++11 и новее допустима форма `[[gnu::...]]`. Это расширение компилятора, не стандарт языка, поэтому переносимость нужно оценивать отдельно.

Чаще всего используются атрибуты, влияющие на инлайнинг и размещение кода. `always_inline` заставляет компилятор встроить функцию при оптимизациях (если это технически возможно). Обычно применяется вместе с `inline`. Обратный вариант — `noinline`, запрет встраивания. В embedded это полезно для сохранения точки входа, отладки или контроля размера кода. Атрибут `section("name")` позволяет поместить функцию или переменную в конкретную секцию, например в `.fastcode` или в область, размещаемую линкер-скриптом во Flash или SRAM.

Для контроля выравнивания и упаковки структур применяются `aligned(n)` и `packed`. Первый задаёт выравнивание объекта, второй отключает добавление паддинга в структуре. `packed` нужно использовать осторожно: на некоторых архитектурах невыравненный доступ приводит к исключениям или падению производительности.

Для управления видимостью символов при линковке используется `visibility("default")`, `visibility("hidden")` и т.п. Это особенно актуально при сборке shared-библиотек. Скрытая видимость уменьшает экспортируемую таблицу символов и ускоряет загрузку.

В системном и embedded-коде часто применяются атрибуты жизненного цикла: `constructor` и `destructor`. Они помечают функции, которые должны выполниться до входа в `main` и после выхода из него соответственно. Это механизм на уровне линкера/загрузчика.

Для работы с прерываниями и низкоуровневым ABI используются `interrupt`, `naked` и `weak`. `interrupt` генерирует пролог/эпилог в соответствии с соглашением обработчика прерывания конкретной архитектуры. `naked` отключает автоматическую генерацию пролога/эпилога — функция полностью управляется вручную (обычно встраиваемым ассемблером). `weak` объявляет символ как слабый; его можно переопределить в другом модуле. Это стандартный приём для векторных таблиц и переопределяемых хук-функций.

Для оптимизации и анализа важны атрибуты, описывающие поведение функции. `pure` сообщает, что функция не имеет побочных эффектов, кроме чтения памяти. `const` — более строгий вариант: функция зависит только от аргументов и не читает глобальную память. Это даёт компилятору возможность агрессивных оптимизаций и устранения повторных вызовов. `malloc` говорит, что функция возвращает новый, уникальный указатель без алиасов, что улучшает анализ зависимостей. `nonnull` позволяет указать, какие параметры не должны быть `NULL`, что усиливает диагностику и оптимизацию.

Для контроля предупреждений и диагностики используется `format(printf, ...)`, который заставляет компилятор проверять соответствие форматной строки и аргументов. Это критично для собственных логирующих функций. Также применяются `deprecated` для пометки устаревших API и `unused` для подавления предупреждений о неиспользуемых переменных или функциях.

Для тонкой настройки оптимизации есть `optimize("O3")` или `target("arch=...")`, позволяющие задать уровень оптимизации или расширения процессора для конкретной функции. Это используется редко, но полезно при критичных вычислительных участках.

В современных версиях GCC большинство этих атрибутов можно писать в форме `[[gnu::always_inline]]`, что синтаксически чище и лучше сочетается со стандартными атрибутами вроде `[[nodiscard]]`.


Принудительный инлайнинг и запрет инлайнинга. Это применяют в коротких функциях доступа к регистрам или, наоборот, когда нужно сохранить точку входа для отладки.
```C
static inline __attribute__((always_inline))
uint32_t reg_read(volatile uint32_t* addr)
{
    return *addr;
}

__attribute__((noinline))
void slow_path(void)
{
    // тяжёлая ветка, удобна как отдельная функция
}
```

Размещение в конкретной секции. Используется вместе с линкер-скриптом, например для размещения обработчиков в отдельной области Flash или для функций, которые должны исполняться из RAM.
```C
__attribute__((section(".ram_code")))
void critical_routine(void)
{
    // код, исполняемый из SRAM
}
```

Слабые символы. Классический приём в embedded — объявить обработчик прерывания как `weak`, чтобы пользователь мог его переопределить.
```C
__attribute__((weak))
void Default_Handler(void)
{
    while (1) {}
}
```

Выравнивание и упаковка структур. Актуально при работе с аппаратными регистрами или сетевыми протоколами.
```C
struct __attribute__((packed)) EthHeader
{
    uint8_t  dst[6];
    uint8_t  src[6];
    uint16_t type;
};

uint8_t buffer[1024] __attribute__((aligned(32)));
```

Атрибуты для оптимизатора: `pure`, `const`, `malloc`. Они помогают компилятору корректно анализировать зависимости.
```C
__attribute__((pure))
int sum_array(const int* a, size_t n)
{
    int s = 0;
    for (size_t i = 0; i < n; ++i)
        s += a[i];
    return s;
}

__attribute__((malloc))
void* my_alloc(size_t size);
```

Проверка форматных строк. Если вы пишете собственный логгер, это обязательная практика.
```C
#include <stdarg.h>
#include <stdio.h>

__attribute__((format(printf, 1, 2)))
void log_printf(const char* fmt, ...)
{
    va_list args;
    va_start(args, fmt);
    vprintf(fmt, args);
    va_end(args);
}
```

Обработчик прерывания (архитектурно-зависимо). На некоторых платформах GCC поддерживает атрибут `interrupt`, который формирует корректный пролог и эпилог.
```C
void __attribute__((interrupt))
Timer_IRQHandler(void)
{
    // код ISR
}
```

Функции-конструкторы, выполняемые до `main`. Это используется в рантайме или при инициализации подсистем.
```C
__attribute__((constructor))
static void init_subsystem(void)
{
    // выполняется до main()
}
```

Ограничение видимости символа при сборке shared-библиотеки.
```C
__attribute__((visibility("hidden")))
void internal_helper(void)
{
}
```

Локальная настройка оптимизации или целевой архитектуры для конкретной функции.
```C
__attribute__((optimize("O3")))
int fast_compute(int x)
{
    return x * x;
}

__attribute__((target("sse4.2")))
int simd_version(int x)
{
    return x;
}
```

Препроцессор в C/C++ — это этап трансляции, который выполняется до компиляции. Он работает с текстом, а не с типами и не с синтаксическим деревом. Его задача — включение файлов, макроподстановки и условная компиляция. Всё происходит на уровне замены токенов.
Главные директивы начинаются с `#`. Они обрабатываются до того, как код попадёт в компилятор.



## `#include`
Подключает содержимое другого файла буквально текстовой вставкой.
```C
#include <vector>
#include "my_header.h"
```
Угловые скобки — поиск в системных путях.  
Кавычки — сначала текущая директория, затем системные пути.
Важно понимать: это не «импорт», а текстовая подстановка. Поэтому include-guard или `#pragma once` обязательны.



## `#define`
Определяет макрос. Это простая текстовая замена.
Простейший пример:
`#define PI 3.1415926535`
После препроцессора все `PI` заменятся на число.

## Вычисляемые выражения
Макрос может содержать выражение:
`#define BUFFER_SIZE (1024 * 4)`
Важно ставить скобки, иначе при подстановке возможны ошибки приоритетов.
Плохо:
`#define SQUARE(x) x * x`
Использование:
`SQUARE(2 + 3)  // превратится в 2 + 3 * 2 + 3`
Правильно:
`#define SQUARE(x) ((x) * (x))`
Теперь подстановка безопаснее.



## Указание типа чисел
Препроцессор не знает типов, но литералы можно писать с суффиксами:
- `10u` — unsigned
- `10UL` — unsigned long
- `10LL` — long long
- `3.14f` — float
- `1.0L` — long double
Пример:
`#define TIMEOUT_MS 1000u`
Это важно, чтобы избежать неявных преобразований.



## Макросы с параметрами
`#define MAX(a, b) ((a) > (b) ? (a) : (b))`
Это текстовая подстановка, не функция. Побочные эффекты опасны:
`MAX(i++, j++)`
Аргументы могут вычисляться несколько раз.
В современном C++ такие вещи лучше делать через `constexpr` или шаблоны.



## Многострочные макросы
Используется `\` для продолжения строки:
```C
#define LOG(msg) \
    do { \
        std::cout << msg << std::endl; \
    } while (0)
```
Конструкция `do { } while(0)` делает макрос синтаксически безопасным внутри `if`.



## Операторы `#` и `##`
`#` — строкизация аргумента:
`#define TO_STRING(x) #x`
`TO_STRING(hello)` → `"hello"`
`##` — склеивание токенов:
`#define CONCAT(a, b) a##b`
`CONCAT(var, 1)` → `var1`
Это используется в метапрограммировании и генерации кода.



## Условная компиляция
Позволяет включать код в зависимости от макросов.
```C
#ifdef DEBUG
    std::cout << "debug\n";
#endif
```
Другие варианты:
```C
#if VERSION > 2
#endif

#ifndef HEADER_H
#define HEADER_H
#endif
```
Доступные директивы:
- `#if`
- `#ifdef`
- `#ifndef`
- `#elif`
- `#else`
- `#endif`
Выражения в `#if` вычисляются препроцессором. Они могут включать арифметику и `defined()`:
`#if defined(DEBUG) && DEBUG_LEVEL > 1 #endif`
Препроцессор понимает только целочисленные выражения.



## Include guards
Классический способ защиты от двойного включения:
```
#ifndef MY_HEADER_H
#define MY_HEADER_H

// содержимое

#endif
```
Альтернатива:
`#pragma once`
Это не стандарт C++, но поддерживается всеми основными компиляторами.



## Предопределённые макросы
Компилятор автоматически определяет:
- `__FILE__`
- `__LINE__`
- `__DATE__`
- `__TIME__`
- `__cplusplus`
Пример:
`std::cout << __FILE__ << ":" << __LINE__;`
Полезно для логирования.

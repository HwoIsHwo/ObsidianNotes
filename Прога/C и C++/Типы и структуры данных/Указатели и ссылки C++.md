Указатели и ссылки в C++ — это механизмы работы с объектами по адресу. Они решают схожие задачи, но имеют разную семантику, ограничения и области применения. Понимание различий принципиально для корректного API и управления временем жизни.

Указатель — это объект, который хранит адрес другого объекта. Он может быть нулевым, может менять значение, может указывать «в никуда», если используется некорректно.
```
int x = 10;
int* p = &x;

std::cout << *p;  // 10
```
`p` — отдельная переменная. Она занимает память. Через `*p` происходит разыменование.

Указатель можно изменить:
```
int y = 20;
p = &y;

p = nullptr;
```
Это делает указатели гибкими, но требует контроля.


Ссылка — это альтернативное имя уже существующего объекта. После инициализации её нельзя переназначить.
```
int x = 10;
int& ref = x;

ref = 20;  // меняет x
```
`ref` не хранит адрес как самостоятельный объект (с точки зрения модели языка). Это просто псевдоним. Она обязана быть инициализирована при создании и не может быть `null`.

Это ключевые различия:
- указатель может быть `nullptr`, ссылка — нет
- указатель можно переназначить, ссылку — нет
- указатель нужно разыменовывать явно, ссылка используется как обычная переменная
### Когда использовать указатель
1. Объект может отсутствовать
2. Нужно менять, на что указывает переменная
3. Работа с динамической памятью
4. Низкоуровневый код

### Когда использовать ссылку
1. Параметр обязателен
2. Нет концепции «отсутствия»
3. Нужно избежать копирования

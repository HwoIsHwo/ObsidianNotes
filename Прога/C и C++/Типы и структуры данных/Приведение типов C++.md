C++ ввёл четыре безопасных явных cast’а для разных ситуаций.

### 1 `static_cast`
- Для **безопасного преобразования типов** в рамках иерархии классов, встроенных типов и совместимых указателей.
- Проверяется на этапе компиляции.
Примеры:
```
int i = 10;
double d = static_cast<double>(i); // int -> double

struct Base { virtual void foo() {} };
struct Derived : Base { void bar() {} };

Base* b = new Derived;
Derived* d_ptr = static_cast<Derived*>(b); // безопасно, если объект действительно Derived
```
Особенности:
- Не удаляет `const`.
- Безопасен для «логически совместимых» типов.
- Быстрее и безопаснее C-style cast для обычных преобразований.


### 2 `dynamic_cast`
- Для **преобразования указателей или ссылок внутри иерархии с виртуальными функциями**.
- Работает только с полиморфными типами (есть хотя бы один виртуальный метод).
Примеры:
```
Base* b = new Derived;
Derived* d_ptr = dynamic_cast<Derived*>(b); // nullptr, если b не Derived

try {
    Derived& d_ref = dynamic_cast<Derived&>(*b); // бросает std::bad_cast, если не Derived
} catch (std::bad_cast&) {}
```
Особенности:
- Проверка во время выполнения (runtime type checking)
- Используется при полиморфизме, когда тип объекта может быть неизвестен


### 3 `const_cast`
- Для **добавления или снятия const/volatile**
- Других преобразований не делает
Пример:
```
const int x = 42;
int& y = const_cast<int&>(x); // снимает const
y = 10; // UB, если объект действительно const
```
Важно:
- Не стоит менять объект, если он изначально был const → UB.
- Используется чаще всего для API, которые требуют неконстантный указатель, хотя данные на самом деле не изменяются.


### 4 `reinterpret_cast`
- Для **низкоуровневого приведения типов**, часто указателей.
- Позволяет рассматривать битовую последовательность одного типа как другой.
- Очень небезопасно, использовать только когда точно понимаешь последствия.

Примеры:
```
int* p = new int(42);
char* c = reinterpret_cast<char*>(p); // рассматриваем int как массив байт

long l = reinterpret_cast<long>(p);   // указатель → целое
```
Особенности:
- Не меняет представление в памяти, просто интерпретирует биты по-другому.
- UB, если неправильно использовать (например, читать объект как несовместимый тип).

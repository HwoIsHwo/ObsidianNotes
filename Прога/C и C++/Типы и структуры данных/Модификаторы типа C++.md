## `const`
- Делает объект **константным**.
- Запрещает изменение значения после инициализации.
- Может использоваться с переменными, указателями, ссылками, методами класса.
Примеры:
```
const int x = 5;
// x = 10; // ошибка

const int* p = &x; // указатель на const int (данные нельзя менять через p)
int* const q = &x; // константный указатель (нельзя изменить сам указатель, но данные можно)

struct A {
    int val;
    int get() const { return val; } // метод не меняет объект
};
```


## `volatile`
- Сообщает компилятору, что значение переменной **может измениться вне программы** (аппарат, другой поток).
- Компилятор **не оптимизирует доступ** к такой переменной: каждый раз будет читать/писать в память.
Примеры:
```
volatile int flag = 0;

// Ждём установки флага другим потоком или устройством
while(flag == 0) { } // без volatile цикл мог бы быть оптимизирован в бесконечное ожидание
```
Используется для:
- регистров аппаратуры в embedded-системах
- флагов синхронизации между потоками (не для сложной логики, только примитивный сигнал)


## `mutable`
- Позволяет **изменять поле объекта** даже если объект объявлен `const`.
- Часто используется для кэширования внутри `const` методов.
Пример
```
struct Cache {
    mutable int last = 0;

    int compute() const {
        last++;
        return 42;
    }
};

const Cache c;
int x = c.compute(); // last увеличится, хотя объект const
```


## `static` (в контексте переменных)
- Для **локальных переменных**: сохраняет значение между вызовами функции.
- Для **членов класса**: общая переменная для всех объектов.
Пример:
```
void foo() {
    static int counter = 0; // сохраняется между вызовами
    counter++;
    std::cout << counter << "\n";
}

struct A {
    static int shared; // одно для всех экземпляров
};
```


## `extern`
- Говорит, что переменная определена **в другом месте** (обычно в другом файле).
```
extern int x; // используем переменную, определённую в другом модуле
```


## `thread_local` (C++11+)
- Создаёт **переменную отдельную для каждого потока**.
- Каждому потоку свой экземпляр.
```
thread_local int counter = 0;
```

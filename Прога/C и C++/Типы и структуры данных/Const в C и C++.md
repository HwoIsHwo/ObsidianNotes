`const` — это квалификатор типа. Он запрещает изменение объекта через данное имя (или через данный тип), но не делает объект «магически неизменяемым» во всех смыслах. Важно понимать, **что именно является константным** и на каком уровне.


1. Базовый случай
```C
const int x = 10;
x = 20; // ошибка
```
`x` — неизменяемый объект. Любая попытка записи через `x` — ошибка компиляции.
Это ограничение на уровне типа. Компилятор может разместить такой объект в read-only секции, но стандарт этого не гарантирует.


2. `const` и указатели — три разных варианта
```C
const int *p;      // указатель на const int
int * const p2;    // const указатель на int
const int * const p3;
```
### `const int *p`
Нельзя менять значение по адресу:
```C
*p = 5; // ошибка
p = &other; // можно
```
### `int * const p2`
Нельзя менять сам указатель:
```C
p2 = &other; // ошибка
*p2 = 5;     // можно
```
### `const int * const p3`
Нельзя ни переназначить указатель, ни менять объект через него.
Главное правило: `const` относится к тому, что слева от него.  
Если слева ничего нет — к тому, что справа.


3. `const` и массивы
```C
const int arr[10];
```
Элементы массива неизменяемы.
```C
int a = 5;
const int *p = &a;
a = 6;   // допустимо
```


4. `const` в аргументах функции
### Передача по значению
```C
void f(const int x);
```
Бессмысленно для вызывающего кода. Это лишь запрет на изменение локальной копии внутри функции.

### Передача указателя
```C
void f(const int *p);
```
Правильный способ показать, что функция не будет менять данные.
Это важная часть API-дизайна.


5. `const` vs `#define`
```C
#define N 10 
const int N = 10;
```
Разница принципиальная:
- `#define` — текстовая подстановка (препроцессор).
- `const` — объект с типом.
`const` участвует в типовой системе, имеет адрес, obeys scope rules.


6. `const` и `volatile`
```C
const volatile int reg;
```
- `const` — запрещает запись через данный тип
- `volatile` — запрещает оптимизатору убирать обращения
Типичный пример — memory-mapped регистры.

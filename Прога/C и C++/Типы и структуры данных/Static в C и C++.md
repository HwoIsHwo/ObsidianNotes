`static` в C и C++ — это не «одна вещь», а набор разных смыслов, зависящих от контекста. Он может влиять либо на **время жизни объекта**, либо на **линковку (visibility)**, либо на **членство в классе** (в C++). Разбирать нужно по месту использования.


1. `static` у глобальных переменных и функций (внутренняя линковка)
```C
static int counter = 0;

static void helper(void) {
    // ...
}
```
Здесь `static` означает **internal linkage**: символ виден только внутри текущей единицы трансляции (.c/.cpp файла).
Это не про время жизни — глобальные объекты и так живут всю программу. Это именно ограничение области видимости на уровне линковки.
Практический смысл:
- избегаем конфликтов имён между файлами,
- скрываем реализацию модуля.
Если убрать `static`, символ станет внешним (external linkage) и может быть использован через `extern` из других файлов.


2. `static` у локальных переменных (статическое время жизни)
```C
void f(void) {
    static int x = 0;
    x++;
}
```
Здесь `static` меняет **время жизни**:
- без `static`: переменная автоматическая (на стеке), создаётся при входе в функцию;
- со `static`: создаётся один раз, хранится в static storage (обычно .data/.bss), живёт до завершения программы.

Особенности:
- инициализация выполняется один раз;
- значение сохраняется между вызовами;
- область видимости остаётся локальной (доступна только внутри блока).
Это по сути «глобальная переменная с ограниченной областью видимости».


3. `static` в C++ внутри класса (статический член)
```C++
class A {
public:
    static int count;
    static void foo();
};
```
Здесь смысл другой:
- `static` поле не принадлежит конкретному объекту;
- существует в единственном экземпляре;
- хранится отдельно от объектов класса.
Определяется вне класса:
```C++
int A::count = 0;
```
Статический метод:
- не имеет `this`;
- может обращаться только к статическим членам класса.


## 4. `static` в заголовочных файлах
Если написать в `.h`:
```C
static int x = 5;
```
и подключить этот заголовок в 5 `.c` файлов — каждый получит **свою отдельную копию `x`**.  
Потому что `static` даёт внутреннюю линковку.
Это часто источник скрытых ошибок.

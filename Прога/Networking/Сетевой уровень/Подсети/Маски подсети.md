Вы, возможно, помните, что IP-адрес представляет собой 32-разрядное число. В мире без подсетей определенное количество этих разрядов используется для идентификатора сети, а другое — для идентификатора хоста. В мире с подсетями <u>некоторые разряды</u>, которые обычно составляют идентификатор хоста, на самом деле <u>используются для идентификатора подсети.</u>

![[Pasted image 20230918221254.png]]

Поскольку все три идентификатора могут быть представлены одним IP-адресом, теперь у нас есть одно 32-разрядное число, которое может быть точно доставлено по многим различным сетям.

На уровне Интернета основные маршрутизаторы обрабатывают только сетевой идентификатор и используют его для отправки датаграммы на соответствующий шлюзовой маршрутизатор этой сети. Затем этот шлюзовой маршрутизатор получает дополнительную информацию, которую он может использовать для отправки датаграммы на конечный компьютер или следующий маршрутизатор на пути к нему. Наконец, идентификатор хоста используется последним маршрутизатором для доставки датаграммы на конечный компьютер-получатель.

Идентификаторы подсетей рассчитываются с помощью так называемой <u>маски подсети</u>. Как и IP-адрес, маски подсетей представляют собой 32-битные числа, которые обычно записываются в виде четырех октетов в десятичной системе счисления.

Давайте снова поработаем с IP-адресом 9.100.100.100. Вы, возможно, помните, что каждая часть IP-адреса представляет собой октет, то есть состоит из восьми битов. Число 9 в двоичной системе счисления — это просто 1001. Но поскольку каждый октет должен состоять из восьми битов, нам нужно добавить в начало несколько нулей.

![[Pasted image 20230918221853.png]]

<u>Маска подсети</u> — это двоичное число, состоящее из двух частей. Начальная часть, которая и является маской, представляет собой последовательность единиц. За ней следуют только нули. Маска подсети, то есть часть числа, состоящая из единиц, указывает, что можно игнорировать при вычислении идентификатора хоста. Часть, состоящая из нулей, указывает, что нужно сохранить.

![[Pasted image 20230918222033.png]]

<u>Цель маски</u> или части, состоящей из одних единиц, заключается в том, чтобы сообщить маршрутизатору, какая часть IP-адреса является идентификатором подсети.

Для 9.100.100.100 это сеть класса A. Мы знаем, что это только первый октет. Таким образом, у нас остаются последние три октета. Возьмем эти оставшиеся октеты и представим их рядом с маской подсети в двоичной форме. <u>Числа в оставшихся октетах, которые имеют соответствующий единицу в маске подсети, являются идентификатором подсети</u>. Числа в оставшихся октетах, которые имеют соответствующий ноль, являются идентификатором хоста.

Здесь уместно отметить, что, как правило, подсеть может содержать только на два меньше, чем общее количество доступных идентификаторов хостов. Опять же, используя маску подсети 255.255.255.0, мы знаем, что октет, доступный для идентификаторов хостов, может содержать числа от 0 до 255, но ноль обычно не используется, а 255 обычно зарезервирован как адрес широковещательной передачи для подсети. Это означает, что на самом деле для назначения хосту доступны только числа от 1 до 254.

Теперь давайте посмотрим на маску подсети, границы которой не проходят по всему октету или восьми битам адреса. Маска подсети 255.255.255.224 будет преобразована в 27 единиц, за которыми следуют пять нулей. Это означает, что у нас есть пять битов пространства идентификатора хоста или в общей сложности 32 адреса. Допустим, мы имеем дело с нашим старым знакомым, 9.100.100.100 с маской подсети 255.255.255.224. Поскольку эта маска подсети представляет собой 27 единиц, за которыми следуют пять нулей, более быстрый способ обозначить это — использовать обозначение /27. Весь IP-адрес и маска подсети могут быть записаны как 9.100.100.100/27.

Уровни абстракции и организация тестов в unit-тестировании определяют, насколько тестовая база будет поддерживаемой, масштабируемой и понятной. Проблема обычно возникает не в написании отдельных тестов, а в их накоплении: без структуры тесты быстро превращаются в несвязанный набор проверок.
Существует несколько уровней абстракции, которые условно можно выделить.

---
Самый нижний уровень — отдельный тест (test case). Это минимальная единица, проверяющая одно конкретное поведение. Такой тест должен быть атомарным и самодостаточным: он задаёт входные данные, вызывает код и проверяет результат.
```cpp
TEST(StackTest, PushIncreasesSize) {
    Stack s;
    s.push(10);
    EXPECT_EQ(s.size(), 1);
}
```
Здесь тест описывает одно свойство: после push размер увеличивается.

---
Следующий уровень — группа тестов (test suite). Это логическая группировка тестов, обычно по классу или модулю. В gtest это первый аргумент макроса `TEST`.
```cpp
TEST(StackTest, PushIncreasesSize) { ... }
TEST(StackTest, PopDecreasesSize) { ... }
```
Все тесты, относящиеся к `Stack`, объединяются в один suite. Это упрощает навигацию и фильтрацию.

---
Далее идёт уровень fixture. Это абстракция, позволяющая вынести общую инициализацию и состояние. Она устраняет дублирование и делает тесты более компактными.
```cpp
class StackTest : public ::testing::Test {
protected:
    Stack s;

    void SetUp() override {
        s.push(1);
        s.push(2);
    }
};

TEST_F(StackTest, InitialSize) {
    EXPECT_EQ(s.size(), 2);
}

TEST_F(StackTest, PopWorks) {
    EXPECT_EQ(s.pop(), 2);
}
```
Fixture задаёт общий контекст, в котором выполняются тесты. Это уже уровень абстракции над отдельными тестами.

---
Следующий уровень — параметризованные тесты. Здесь абстрагируется не только окружение, но и входные данные. Один тест описывает поведение для множества случаев.
```cpp
class EvenTest : public ::testing::TestWithParam<int> {};

TEST_P(EvenTest, Works) {
    int v = GetParam();
    EXPECT_EQ(v % 2 == 0, isEven(v));
}

INSTANTIATE_TEST_SUITE_P(
    Cases,
    EvenTest,
    ::testing::Values(1, 2, 3, 4)
);
```
Это уровень абстракции над тест-кейсами, позволяющий избежать дублирования логики.

---
Ещё выше — уровень вспомогательных функций и DSL-подобных обёрток. Когда тесты становятся сложными, в них начинают появляться повторяющиеся конструкции, которые выносятся в отдельные функции.
```cpp
void checkPush(Stack& s, int value, int expectedSize) {
    s.push(value);
    EXPECT_EQ(s.size(), expectedSize);
}

TEST(StackTest, MultiplePush) {
    Stack s;
    checkPush(s, 1, 1);
    checkPush(s, 2, 2);
}
```
Это снижает дублирование и повышает читаемость, но при чрезмерном использовании может скрывать логику теста.

Иногда вводится дополнительная иерархия:
- unit
- integration
- e2e

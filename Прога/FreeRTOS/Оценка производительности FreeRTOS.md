Первый подход - использовать в задаче простоя какую-либо инкрементируемую переменную, для определения времени простоя. Либо переключение портов ввода/вывода, для отслеживания работы задачи простоя при помощи логического анализатора.

Второй подход - использование таймера с более высокой частотой, чем частота работы планировщика (частота работы ОС), для оценки времени работы задачи простоя.

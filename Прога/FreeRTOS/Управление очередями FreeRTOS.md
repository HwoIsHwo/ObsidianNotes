Формально очередь - фиксированного размера хранилище данных, содержащее конечное число элементов. Чаще всего используется в качестве буферов FIFO.

На примере очередь используется для обмена данными между задачами:
![[{28AD3ED6-838A-4917-9E85-556B3D190A15}.png]]
Задача 1 каждый вызов помещает элемент в очередь (2 вызова на иллюстрации, для значений 9 и 3). Когда задача 2 получает управление, она читает первый элемент в очереди (это будет 9 по принципу FIFO).
![[{3B9EEC3F-2FED-450E-925F-87A7A898F366}.png]]
Первый элемент прочитан -  вся очередь смещается на один вперед, а первый элемент удаляется. 

В рамках FreeRTOS в очередь помещаются непосредственно сами данные, а не ссылка на них. То есть происходит копирование данных. ОС берет на себя все операции по выделению памяти для элементов очереди.

Свойства очередей:
- многозадачный доступ. Очередь - самостоятельный объект. Любая задача/обработчик прерываний, знающие о существовании очереди могут получить к ней доступ. 
- Любое количество задач могут помещать данные в очередь и любое количество задач могут эти данные считывать.
- таймаут чтения из очереди.  При чтении из очереди задача может использовать время таймаута, отводимое на операцию чтения. Это время в течении которого задача будет заблокирована в ожидании доступности данных из очереди.
- задача будет переведена в состояние "готова" по истечении таймаута до того как данные станут доступны.
- так как у очереди может быть несколько читателей, ожидающих данные, она может блокировать несколько задач.
- также есть таймаут на отправку данных в очередь
- очереди можно группировать

Функции API FreeRTOS:
Создание очереди
![[{CC9A5463-A8EF-4C09-9606-EFAB9CA9DFEC}.png]]
Параметры:
- количество элементов
- размер в байтах каждого элемента

Отправка данных в очередь:
![[{46C244BC-62A5-4656-B668-249B99BE9DC9}.png]]
Нельзя использовать в обработчиках прерываний.
Параметры:
- дескриптор очереди
- указатель на помещаемые данные
- таймаут (сколько можем ждать освобождения места для отправки данных.)

Чтение данных из очереди:
![[{E5288C75-6FF1-4450-8E7B-849B68782D0C}.png]]
Первая функция считывает элемент и удаляет его из очереди. Параметры:
- дескриптор очереди
- указатель на элемент для копирования данных из очереди
- таймаут
Вторая функция - позволяет узнать сколько элементов в очереди.

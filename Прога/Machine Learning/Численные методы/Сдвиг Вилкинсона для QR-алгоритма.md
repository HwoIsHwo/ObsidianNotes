Сдвиг Вилкинсона является приёмом ускорения QR-алгоритма, основанным на более точном учёте локальной спектральной структуры матрицы на завершающих стадиях итерационного процесса. Его идея заключается в том, чтобы на каждом шаге QR-алгоритма не работать с исходной матрицей напрямую, а предварительно вычитать из неё специально подобранный скаляр, близкий к одному из собственных значений. Такое смещение существенно уменьшает величины поддиагональных элементов и тем самым ускоряет сходимость.

В качестве сдвига используется информация из нижнего правого блока матрицы, поскольку именно он сходится последним. Для вещественного QR-алгоритма этот блок имеет размер либо 1×1, либо 2×2. В случае 2×2 блока сдвиг Вилкинсона выбирается как собственное значение этого блока, ближайшее к его правому нижнему элементу. Такой выбор не случаен: он обеспечивает численную устойчивость формулы и гарантирует, что введённый сдвиг отражает реальное локальное поведение спектра.

Геометрически сдвиг Вилкинсона можно интерпретировать как временный перенос начала координат в спектральном пространстве, при котором QR-алгоритм «видит» матрицу уже почти приведённой к блочно-треугольному виду. После выполнения QR-разложения сдвиг возвращается обратно, и итоговое преобразование остаётся ортогонально подобным исходной матрице. В результате сдвиг Вилкинсона не изменяет собственные значения, но позволяет достигать квадратичной или даже кубической сходимости на завершающем этапе вычислений.


## Алгоритм
```
    int n = R.rows;
    if (n < 2)
        return R.data[n-1][n-1];

    double a = R.data[n-2][n-2];
    double b = R.data[n-2][n-1];
    double c = R.data[n-1][n-2];
    double d = R.data[n-1][n-1];

    double delta = (a - d) * 0.5;
    double sign = (delta >= 0.0) ? 1.0 : -1.0;

    double mu = d - sign * (b * c) / (std::abs(delta) + std::sqrt(delta*delta + b*c));

    return mu;
```

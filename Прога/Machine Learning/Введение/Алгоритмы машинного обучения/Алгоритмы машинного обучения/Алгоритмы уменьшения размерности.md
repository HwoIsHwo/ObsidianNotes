Алгоритмы снижения размерности предназначены для обработки точки данных и создания более эффективного её представления. Более конкретно, они предназначены для обучения отображению точек данных высокой размерности в пространство, где их можно точно описать с помощью меньшего количества признаков: другими словами, для сокращения количества измерений, необходимых для эффективного представления данных.

Снижение размерности часто выполняется на этапе предварительной обработки данных, помогая снизить сложность и уровень шума в данных для улучшения прогнозов или снижения вычислительных затрат. Это также важный этап моделирования скрытого пространства набора данных: сжатого (меньшая размерность) представления данных, сохраняющего только подмножество признаков, наиболее релевантных решаемой задаче. Другие распространённые примеры снижения размерности включают сжатие и визуализацию данных.

- Анализ главных компонент (PCA) упрощает сложные наборы данных, суммируя исходные переменные данных, многие из которых часто коррелируют друг с другом и, следовательно, являются несколько избыточными, в виде меньшего подмножества некоррелированных переменных, каждая из которых представляет собой линейную комбинацию исходных переменных. Более конкретно, алгоритм отдаёт приоритет главным компонентам данных: линейным комбинациям переменных, которые имеют наибольшую дисперсию по сравнению с другими линейными комбинациями.
- Стохастическое встраивание соседних элементов с t-распределением (t-SNE) — это нелинейный алгоритм снижения размерности, обычно используемый для визуализации данных. Он используется почти исключительно для представления данных в двух- или трёхмерном пространстве, с основной целью — гарантировать, что точки данных, расположенные близко друг к другу в многомерном пространстве, останутся близкими друг к другу и в новом маломерном пространстве.
- Автоэнкодеры — это тип архитектуры нейронной сети кодер-декодер, обучаемой с помощью алгоритма, который чаще можно назвать самоконтролируемым обучением (поскольку его цель — минимизировать функцию потерь), но при этом они выполняют снижение размерности немаркированных данных, в данном случае моделируя их скрытое пространство. Кодер состоит из ряда постепенно уменьшающихся слоёв, заставляя входные данные проходить через «узкое место», которое «сжимает» данные во всё меньшее число измерений, прежде чем они достигнут декодера. Декодер, состоящий из ряда постепенно увеличивающихся слоёв, затем должен реконструировать исходные данные, используя это сжатое представление, с целью минимизации потерь при реконструкции. Это заставляет кодер учиться извлекать и передавать только ту информацию, которая наиболее способствует точной реконструкции исходных входных данных.

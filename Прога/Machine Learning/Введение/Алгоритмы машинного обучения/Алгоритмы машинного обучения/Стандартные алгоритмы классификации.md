Наивные байесовские классификаторы работают по логике теоремы Байеса, которая, по сути, представляет собой математическую формулировку идеи о том, что информация о более поздних событиях (например, результатах) может быть использована для обновления понимания более ранних событий (например, входных данных). Другими словами, модель определяет относительную важность заданной входной переменной на основе её корреляции с конкретными результатами. Её одноимённое «наивное» предположение заключается в том, что все признаки, влияющие на классификацию, независимы друг от друга. Это упрощение делает алгоритм быстрым и эффективным для простых задач, таких как обнаружение спама.

Логистическая регрессия адаптирует алгоритм линейной регрессии для решения задач бинарной классификации, подавая взвешенную сумму входных признаков в сигмоидальную функцию, которая преобразует любые входные данные в значение от 0 до 1. Полученное значение можно интерпретировать как вероятность наступления заданного события — в данном случае, конкретной классификации.

Алгоритмы K-ближайших соседей (KNN) классифицируют точки данных на основе их близости в пространстве векторного вложения к другим, уже классифицированным, то есть помеченным, точкам данных, предполагая, что похожие точки данных могут быть найдены рядом друг с другом. Число k определяет количество учитываемых соседних точек данных: например, в алгоритме KNN, где k = 5, входная точка данных сравнивается с 5 ближайшими соседями и классифицируется в соответствии с той категорией, которая наиболее представлена ​​среди этих 5 соседних точек данных.

Машины опорных векторов (SVM) — это мощные модели, которые выполняют бинарную классификацию, но также могут быть адаптированы для задач многоклассовой классификации. Цель алгоритма SVM заключается не в том, чтобы напрямую научиться категоризировать точки данных: вместо этого его цель — изучить оптимальную границу принятия решений для разделения двух категорий помеченных точек данных, чтобы затем классифицировать новые точки данных на основе того, с какой стороны границы они находятся. Алгоритм SVM определяет эту границу как гиперплоскость, максимизирующую зазор (или разрыв) между точками данных противоположных классов. Эта концепция аналогична предположению о низкой плотности в полуконтролируемом обучении. Логично, что единственные точки данных, которые могут поддерживать вычисление этой гиперплоскости, — это точки данных каждого класса, наиболее близкие к границе. Векторные представления этих точек данных, смежных с границей, поэтому называются опорными векторами.

`.text`, `.data` и `.bss` — это секции (segments/sections) исполняемого файла, которые определяют, как программа размещается в памяти. Они формируются компилятором и компоновщиком и потом загружаются в память операционной системой или стартовым кодом в embedded-системах.



### `.text`
`.text` содержит машинный код программы — инструкции процессора. Обычно эта секция помечена как read-only и executable. В большинстве систем попытка записи в неё приводит к ошибке защиты памяти.
Иногда в `.text` также попадают строковые литералы и другие неизменяемые константы (хотя формально они могут лежать в `.rodata`, которая часто размещается рядом).
Зачем это нужно:  
разделение кода и данных повышает безопасность (нельзя случайно изменить код) и позволяет системе оптимально отображать память, например, разделять один и тот же код между процессами.



### `.data`
`.data` содержит глобальные и статические переменные с явной инициализацией ненулевыми значениями.
Пример:
```C
int x = 42;
static int y = 7;
```
Эти переменные окажутся в `.data`, потому что им нужно сохранить конкретное начальное значение. Это значение записывается в исполняемый файл. При запуске программы загрузчик копирует содержимое `.data` в оперативную память.
Зачем это нужно:  
инициализированные данные должны быть сохранены в образе программы, чтобы при старте они получили правильные значения.



### `.bss`
`.bss` (Block Started by Symbol) содержит глобальные и статические переменные без явной инициализации или инициализированные нулём.
Пример:
```C
int a;
static int b;
int c = 0;
```
Все они попадут в `.bss`.
Главная особенность — в исполняемом файле для них не хранится массив нулей. Линкер просто указывает размер области. При старте программы эта область в памяти обнуляется. Поэтому `.bss` увеличивает потребление RAM, но почти не увеличивает размер файла.
Зачем это нужно:  
экономия места в бинарнике и ускорение загрузки — нет смысла хранить мегабайты нулей.



### Как это работает при запуске
В обычной ОС загрузчик:
1. Загружает `.text` в память (обычно только для чтения).
2. Загружает `.data` и копирует инициализированные значения.
3. Выделяет область под `.bss` и заполняет её нулями.
В embedded-системах это делает стартовый код до вызова `main()`:  
`.data` копируется из Flash в RAM, `.bss` обнуляется вручную.



### Общая идея
Разделение на `.text`, `.data` и `.bss` нужно для:
- чёткого разделения кода и данных,
- экономии места в исполняемом файле,
- правильной инициализации глобальных объектов,
- возможности управлять размещением памяти (особенно в embedded).
Это фундаментальная часть модели выполнения программы на уровне компиляции и линковки.

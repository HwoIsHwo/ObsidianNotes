Эффективность алгоритма зависит от времени, объема памяти и других ресурсов, необходимых для его выполнения. Эффективность измеряется с помощью асимптотических обозначений.
Алгоритм может демонстрировать разную производительность для разных типов входных данных. С увеличением размера входных данных производительность будет меняться.
Изучение изменения производительности алгоритма при изменении размера входных данных называется асимптотическим анализом.


---

Асимптотические обозначения — это математические обозначения, используемые для описания времени выполнения алгоритма, когда входные данные стремятся к определенному или предельному значению. 
Например: в алгоритме сортировки пузырьком, когда входной массив уже отсортирован, время, затрачиваемое алгоритмом, является линейным, то есть наилучшим случаем. Но когда входной массив находится в обратном порядке, алгоритму требуется максимальное время (квадратичное) для сортировки элементов, то есть наихудшим случаем. Когда входной массив не отсортирован и не находится в обратном порядке, алгоритму требуется среднее время. Эти значения обозначаются с помощью асимптотических обозначений.

Существует три основных асимптотических обозначения:
- Обозначение «О большое»
- Обозначение «Омега»
- Обозначение «Тета»


---

## Обозначение «Большая О» (O-обозначение)
Обозначение «Большая О» представляет собой верхнюю границу времени выполнения алгоритма. Таким образом, оно показывает сложность алгоритма в худшем случае.
![[Pasted image 20260228193222.png]]
O(g(n)) = { f(n): существуют положительные константы c и n0
такие, что 0 ≤ f(n) ≤ cg(n) для всех n ≥ n0 }

Вышеприведенное выражение можно описать как функцию f(n), принадлежащую множеству O(g(n)), если существует положительная константа c такая, что она находится между 0 и cg(n) при достаточно больших n.
Для любого значения n время выполнения алгоритма не превышает время, обеспечиваемое O(g(n)).
Поскольку оно показывает время выполнения алгоритма в наихудшем случае, оно широко используется для анализа алгоритмов, так как нас всегда интересует наихудший сценарий.


---

Омега-обозначение (Ω-обозначение)
Омега-обозначение представляет собой нижнюю границу времени выполнения алгоритма. Таким образом, оно показывает наилучшую сложность алгоритма.

![[Pasted image 20260228193343.png]]
Ω(g(n)) = { f(n): существуют положительные константы c и n0
такие, что 0 ≤ cg(n) ≤ f(n) для всех n ≥ n0 }

Приведенное выше выражение можно описать как функцию f(n), принадлежащую множеству Ω(g(n)), если существует положительная константа c такая, что она лежит выше cg(n) при достаточно больших n.

Для любого значения n минимальное время, необходимое для работы алгоритма, определяется выражением Omega Ω(g(n)).


---

Тета-обозначение (Θ-обозначение)
Тета-обозначение охватывает функцию сверху и снизу. Поскольку оно представляет собой верхнюю и нижнюю границы времени выполнения алгоритма, оно используется для анализа сложности алгоритма в среднем случае.
![[Pasted image 20260228193449.png]]
Для функции g(n) Θ(g(n)) определяется соотношением:
Θ(g(n)) = { f(n): существуют положительные константы c1, c2 и n0
такие, что 0 ≤ c1g(n) ≤ f(n) ≤ c2g(n) для всех n ≥ n0 }

Приведенное выше выражение можно описать как функцию f(n), принадлежащую множеству Θ(g(n)), если существуют положительные константы c1 и c2 такие, что она может находиться между c1g(n) и c2g(n) для достаточно больших n.
Если функция f(n) находится где угодно между c1g(n) и c2g(n) для всех n ≥ n0, то f(n) называется асимптотически точной границей.

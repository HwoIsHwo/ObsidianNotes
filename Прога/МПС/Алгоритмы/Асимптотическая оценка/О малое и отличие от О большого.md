 Вкратце, оба они являются асимптотическими обозначениями, которые определяют верхние границы для функций и времени выполнения алгоритмов.

Однако разница заключается в том, что обозначение «большое О» может быть асимптотически точным, в то время как обозначение «маленькое О» гарантирует, что верхняя граница не будет асимптотически точной.

Обозначение «маленькая О» используется для обозначения верхней границы, которая не является асимптотически точной. Формально она определяется следующим образом:

$o(g(n)) = \{f(n)$: для любой положительной константы c существует положительная константа $n_0$ такая, что $0 \le f(n) < cg(n)$ для всех $n \ge n_0\}$.

Заметим, что в этом определении множество функций $\pmb{f(n)}$ строго меньше, чем $\pmb{cg(n)}$, что означает, что обозначение «маленькая О» является более сильной верхней границей, чем обозначение «большая О». Другими словами, обозначение «маленькая O» не позволяет функции f(n) иметь ту же скорость роста, что и g(n).

Интуитивно это означает, что по мере приближения n к бесконечности f(n) становится незначительной по сравнению с g(n). В математических терминах:

$$ \lim_{n\to\infty} \frac{f(n)}{g(n)} = 0 $$
Кроме того, неравенство в определении «маленького О» должно выполняться для любой константы c, тогда как для «большого О» достаточно найти такое c, которое удовлетворяет неравенству.

Если провести аналогию между асимптотическим сравнением f(n) и g(n) и сравнением действительных чисел a и b, то f(n) = O(g(n)) ≈ a ≤ b, а f(n) = o(g(n)) ≈ a < b.

Давайте рассмотрим несколько примеров, чтобы всё стало понятнее.
Для f(n) = 2n + 3 имеем:
$\pmb{f(n) = O(n)}$ но $\pmb{f(n) \ne o(n)}$
$f(n) = O(n^2)$ и $f(n) = o(n^2)$
$f(n) = O(n^3)$ и $f(n) = o(n^3)$
В общем случае, для $f(n) = a_xn^x + a_{x-1}n^{x-1} + ... + a_0$ имеем:
$\pmb{f(n) = O(n^x)}$ но $\pmb{f(n) \ne o(n^x)}$
$f(n) = O(n^{x+1})$ и $f(n) = o(n^{x+1})$
$f(n) = O(n^{x+2})$ и $f(n) = o(n^{x+2})$

Пузырьковая сортировка — это алгоритм сортировки, который сравнивает два соседних элемента и меняет их местами до тех пор, пока они не окажутся в нужном порядке.

Принцип работы пузырьковой сортировки
Предположим, мы пытаемся отсортировать элементы в порядке возрастания.
1. Первая итерация (сравнение и обмен)
	1. Начиная с первого индекса, сравним первый и второй элементы.
	2. Если первый элемент больше второго, они меняются местами.
	3. Теперь сравним второй и третий элементы. Поменяем их местами, если они не совпадают по порядку.
	4. Этот процесс продолжается до последнего элемента.
![[Pasted image 20260304123159.png]]
2. Оставшиеся итерации
	Процесс повторяется и в последующих итерациях. После каждой итерации наибольший элемент среди несортированных элементов помещается в конец.
![[Pasted image 20260304123246.png]]
На каждой итерации сравнение выполняется до последнего несортированного элемента.
![[Pasted image 20260304123304.png]]
Массив считается отсортированным, когда все неотсортированные элементы размещены на своих правильных позициях.
![[Pasted image 20260304123326.png]]
```
bubbleSort(array)
  for i <- 1 to sizeOfArray - 1
    for j <- 1 to sizeOfArray - 1 - i
      if leftElement > rightElement
        swap leftElement and rightElement
end bubbleSort
```

## Оптимизированный алгоритм пузырьковой сортировки
В приведенном выше алгоритме все сравнения выполняются, даже если массив уже отсортирован. Это увеличивает время выполнения.
Для решения этой проблемы можно ввести дополнительную переменную swapped. Значение swapped устанавливается в true, если происходит обмен элементами. В противном случае оно устанавливается в false. После итерации, если обмена не происходит, значение swapped будет false. Это означает, что элементы уже отсортированы, и нет необходимости выполнять дальнейшие итерации. Это сократит время выполнения и поможет оптимизировать пузырьковую сортировку.
```
bubbleSort(array)
  for i <- 1 to sizeOfArray - 1
    swapped <- false
    for j <- 1 to sizeOfArray - 1 - i
      if leftElement > rightElement
        swap leftElement and rightElement
        swapped <- true
    if swapped == false
      break
end bubbleSort
```

1. Временная сложность
	- Сложность в худшем случае: O(n²) Если мы хотим отсортировать в порядке возрастания, а массив расположен в порядке убывания, то возникает худший случай.
	- Сложность в лучшем случае: O(n) Если массив уже отсортирован, сортировка не требуется.
	- Сложность в среднем случае: O(n²) Это происходит, когда элементы массива расположены в перемешанном порядке (ни в порядке возрастания, ни в порядке убывания).
2. Пространственная сложность
	- Пространственная сложность составляет O(1), потому что для перестановки используется дополнительная переменная.
	- В оптимизированном алгоритме пузырьковой сортировки используются две дополнительные переменные. Следовательно, пространственная сложность будет O(2).

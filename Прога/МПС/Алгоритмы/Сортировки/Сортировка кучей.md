Сортировка кучей — популярный и эффективный алгоритм сортировки в компьютерном программировании. Для изучения алгоритма сортировки кучей необходимы знания двух типов структур данных — массивов и деревьев.

Начальный набор чисел, которые мы хотим отсортировать, хранится в массиве, например, \[10, 3, 76, 34, 23, 32], а после сортировки мы получаем отсортированный массив \[3, 10, 23, 32, 34, 76].

Сортировка кучей работает, представляя элементы массива в виде особого вида полного бинарного дерева, называемого кучей.


---
## Взаимосвязь между индексами массива и элементами дерева
Полное бинарное дерево обладает интересным свойством, которое можно использовать для поиска потомков и родителей любого узла. Если индекс любого элемента в массиве равен i, то элемент с индексом 2i+1 станет левым потомком, а элемент с индексом 2i+2 — правым потомком. Кроме того, родитель любого элемента с индексом i определяется нижней границей (i-1)/2.

Понимание этого сопоставления индексов массива с позициями в дереве имеет решающее значение для понимания того, как работает структура данных «куча» и как она используется для реализации сортировки кучей.

## Как преобразовать дерево в кучу
Начиная с полного бинарного дерева, мы можем модифицировать его, превратив в максимальную кучу, применив функцию heapify ко всем нелистовым элементам кучи. Поскольку heapify использует рекурсию, её может быть сложно понять. Поэтому давайте сначала подумаем, как преобразовать дерево в кучу всего с тремя элементами.
![[Pasted image 20260309154120.png]]
Приведённый выше пример демонстрирует два сценария: один, в котором корень является наибольшим элементом, и нам ничего не нужно делать; и другой, в котором корень имел больший дочерний элемент, и нам нужно было произвести обмен, чтобы сохранить свойство максимальной кучи.
Если вы раньше работали с рекурсивными алгоритмами, вы, вероятно, поняли, что это должен быть базовый случай.
Теперь давайте рассмотрим другой сценарий, в котором существует более одного уровня.
![[Pasted image 20260309154151.png]]
Верхний элемент не является максимальной кучей, но все поддеревья являются максимальными кучами. Чтобы сохранить свойство максимальной кучи для всего дерева, нам придётся постоянно сдвигать элемент 2 вниз, пока он не достигнет своего правильного положения.
![[Pasted image 20260309154219.png]]
Таким образом, чтобы сохранить свойство максимальной кучи в дереве, где оба поддерева являются максимальными кучами, нам необходимо многократно применять функцию `heapify` к корневому элементу, пока он не станет больше своих дочерних элементов или не станет листовым узлом.

Мы можем объединить оба этих условия в одной функции `heapify` следующим образом:
```C
void heapify(int arr[], int n, int i) {
  // Find largest among root, left child and right child
  int largest = i;
  int left = 2 * i + 1;
  int right = 2 * i + 2;

  if (left < n && arr[left] > arr[largest])
    largest = left;

  if (right < n && arr[right] > arr[largest])
    largest = right;

    // Swap and continue heapifying if root is not largest
    if (largest != i) {
      swap(&arr[i], &arr[largest]);
      heapify(arr, n, largest);
  }
}
```
Эта функция работает как для базового случая, так и для дерева любого размера. Таким образом, мы можем переместить корневой элемент в правильное положение, чтобы сохранить статус максимальной кучи для любого размера дерева, если поддеревья являются максимальными кучами.

## Построение максимальной кучи
Чтобы построить максимальную кучу из любого дерева, мы можем начать преобразование каждого поддерева в кучу снизу вверх и получить максимальную кучу после применения функции ко всем элементам, включая корневой элемент.
В случае полного дерева первый индекс нелистового узла задается как n/2 - 1. Все остальные узлы после этого являются листовыми узлами и, следовательно, не нуждаются в преобразовании в кучу.
Таким образом, мы можем построить максимальную кучу следующим образом:
```C
    // Build heap (rearrange array)
    for (int i = n / 2 - 1; i >= 0; i--)
      heapify(arr, n, i);
```

## Принцип работы сортировки кучей
1. Поскольку дерево удовлетворяет свойству максимальной кучи, наибольший элемент хранится в корневом узле.
2. Поменять местами: удалить корневой элемент и поместить его в конец массива (n-ю позицию). Поместить последний элемент дерева (кучи) на освободившееся место.
3. Удалить: уменьшить размер кучи на 1.
4. Преобразовать в кучу: снова преобразовать корневой элемент в кучу, чтобы в корне оказался наибольший элемент.
Процесс повторяется до тех пор, пока все элементы списка не будут отсортированы.

## Сложность сортировки кучей
Временная сложность
- Лучший вариант: O(nlog n)
- Худший вариант: O(nlog n)
- Средний вариант: O(nlog n)
Пространственная сложность: O(1)
Стабильность: Нет
Сортировка кучей имеет временную сложность O(nlog n) для всех случаев (лучший, средний и худший).
Давайте разберемся, почему. Высота полного бинарного дерева, содержащего n элементов, равна log n.
Как мы видели ранее, чтобы полностью превратить элемент, поддеревья которого уже являются максимальными кучами, нам нужно постоянно сравнивать этот элемент с его левым и правым потомками и сдвигать его вниз, пока он не достигнет точки, где оба его потомка будут меньше его. В худшем случае нам потребуется переместить элемент из корня в листовой узел, выполнив кратное log(n) сравнений и обменов. На этапе построения максимальной кучи (build_max_heap) мы выполняем это для n/2 элементов, поэтому сложность в худшем случае для этапа построения кучи составляет n/2\*log n ~ nlog n.
На этапе сортировки мы обмениваем корневой элемент с последним элементом и создаем кучу из корневого элемента. Для каждого элемента это снова занимает log n времени в худшем случае, потому что нам, возможно, придется перемещать элемент от корня до листа. Поскольку мы повторяем это n раз, этап сортировки кучей также занимает nlog n.
Кроме того, поскольку этапы построения максимальной кучи и сортировки кучей выполняются один за другим, алгоритмическая сложность не умножается и остается порядка nlog n.
Также, сортировка выполняется с пространственной сложностью O(1). По сравнению с быстрой сортировкой, у нее лучшая сложность в худшем случае (O(nlog n)). Быстрая сортировка имеет сложность O(n^2) в худшем случае. Но в других случаях быстрая сортировка работает быстро. Introsort — это альтернатива пирамидальной сортировке, которая сочетает в себе преимущества быстрой сортировки и пирамидальной сортировки, сохраняя при этом преимущества обеих: скорость пирамидальной сортировки в худшем случае и среднюю скорость быстрой сортировки.

## Применение сортировки кучей
Системы, ориентированные на безопасность, и встроенные системы, такие как ядро ​​Linux, используют сортировку кучей из-за верхней границы времени выполнения O(n log n) и постоянной верхней границы O(1) для вспомогательного хранилища.

Хотя сортировка кучей имеет временную сложность O(n log n) даже в худшем случае, она не имеет большего количества применений (по сравнению с другими алгоритмами сортировки, такими как быстрая сортировка, сортировка слиянием). Однако ее базовая структура данных, куча, может эффективно использоваться, если мы хотим извлечь наименьший (или наибольший) элемент из списка без накладных расходов на поддержание оставшихся элементов в отсортированном порядке. Например, в приоритетных очередях.

Хеш-функция — это функция, которая отображает данные произвольного размера в фиксированное по размеру значение (хеш). В C++ чаще всего речь идёт о хешировании для ассоциативных контейнеров (`std::unordered_map`, `std::unordered_set`), но есть и криптографические применения.
При этом разные входные данные могут дать одинаковый результат — это называется коллизией. Полностью избежать коллизий невозможно, задача — сделать их редкими и равномерно распределёнными.

## Основные свойства хорошей хеш-функции (для контейнеров)
1. Детерминированность — одинаковый вход → одинаковый хеш
2. Быстрота вычисления
3. Равномерное распределение
4. Минимизация коллизий

Важно понимать: для `unordered_map` хеш-функция не обязана быть криптографически стойкой. Её задача — хорошо распределять ключи по бакетам.

## Как используется хеш в `std::unordered_map`
При вставке:
1. Вычисляется `hash(key)`
2. Выбирается бакет: `hash % bucket_count`
3. В бакете происходит проверка равенства ключей
Схематично:
hash(key) →  145345345
                ↓
           145345345 % 16
                ↓
            bucket\[9]
Если в бакете несколько элементов — происходит разрешение коллизии (обычно через список или цепочку узлов).


## std::hash
В стандартной библиотеке есть шаблон `std::hash`.
```
#include <functional>

std::hash<std::string> hasher;
size_t h = hasher("hello");
```
Для стандартных типов уже есть специализации:
- целые числа
- строки
- указатели
- enum
Если использовать пользовательский тип как ключ, нужно определить:
1. оператор `==`
2. специализацию `std::hash`
Пример:
```
struct Point {
    int x;
    int y;

    bool operator==(const Point& other) const {
        return x == other.x && y == other.y;
    }
};

namespace std {
template<>
struct hash<Point> {
    size_t operator()(const Point& p) const {
        return hash<int>()(p.x) ^ (hash<int>()(p.y) << 1);
    }
};
}
```

## Комбинирование хешей
Простое XOR — не всегда лучший вариант. Более корректный способ — использовать микширование:
```
size_t seed = 0;
seed ^= std::hash<int>()(p.x) + 0x9e3779b9 + (seed << 6) + (seed >> 2);
seed ^= std::hash<int>()(p.y) + 0x9e3779b9 + (seed << 6) + (seed >> 2);
```

## Коллизии
Коллизия — разные данные дают одинаковый хеш.
Пример:
hash("abc") == hash("xyz")
Контейнер всё равно корректно работает, потому что после попадания в бакет сравнивает ключи через `==`.
Плохая хеш-функция приводит к:
- деградации производительности
- переходу от O(1) к O(n)


## Криптографические хеш-функции
Это отдельный класс:
- SHA-256
- SHA-3
- BLAKE2
Они обладают свойствами:
- устойчивость к коллизиям
- необратимость
- лавинный эффект
Их используют для:
- цифровых подписей
- хранения паролей (вместе с солью и растяжкой)
- проверки целостности

## Лавинный эффект
Малое изменение входа → сильное изменение хеша.
Пример:
`"hello"  →  A7F2... "hellp"  →  9C3D...`
Для контейнеров это тоже желательно — равномерность распределения.

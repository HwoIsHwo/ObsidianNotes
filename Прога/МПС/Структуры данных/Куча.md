Куча (_heap_) — это специализированная структура данных, представляющая почти полное бинарное дерево, в котором выполняется **свойство кучи**.
Для max-heap: ключ родителя ≥ ключей потомков.  
Для min-heap: ключ родителя ≤ ключей потомков.
Важно: куча **не является** бинарным деревом поиска. Гарантируется только экстремальный элемент в корне. Поиск произвольного элемента остаётся O(n).

Основное назначение — быстрый доступ к минимуму или максимуму.  
Типичные сложности:
вставка — O(log n)  
удаление корня — O(log n)  
получение корня — O(1)  
построение кучи из массива — O(n)

Практически куча почти всегда хранится в массиве. Узлы не выделяются отдельно — структура кодируется индексами.
Для элемента `i`:
```
parent = (i - 1) / 2
left   = 2*i + 1
right  = 2*i + 2
```
Это ключевая причина эффективности: отсутствие указателей и хорошая cache-локальность.

---

Минимальная реализация **max-heap** на C.
```C
#include <stdio.h>
#include <stdlib.h>

#define CAPACITY 128

typedef struct
{
    int data[CAPACITY];
    int size;
} Heap;

void heap_init(Heap* h)
{
    h->size = 0;
}

// обмен элементов
void swap(int* a, int* b)
{
    int t = *a;
    *a = *b;
    *b = t;
}

// просеивание вверх (insert)
void heapify_up(Heap* h, int i)
{
    while (i > 0)
    {
        int parent = (i - 1) / 2;

        if (h->data[parent] >= h->data[i])
            break;

        swap(&h->data[parent], &h->data[i]);
        i = parent;
    }
}

void heap_push(Heap* h, int value)
{
    h->data[h->size] = value;
    heapify_up(h, h->size);
    h->size++;
}

// просеивание вниз (remove)
void heapify_down(Heap* h, int i)
{
    while (1)
    {
        int left  = 2*i + 1;
        int right = 2*i + 2;
        int largest = i;

        if (left < h->size &&
            h->data[left] > h->data[largest])
            largest = left;

        if (right < h->size &&
            h->data[right] > h->data[largest])
            largest = right;

        if (largest == i)
            break;

        swap(&h->data[i], &h->data[largest]);
        i = largest;
    }
}

int heap_pop(Heap* h, int* value)
{
    if (h->size == 0)
        return 0;

    *value = h->data[0];

    h->data[0] = h->data[h->size - 1];
    h->size--;

    heapify_down(h, 0);
    return 1;
}

int heap_top(Heap* h, int* value)
{
    if (h->size == 0)
        return 0;

    *value = h->data[0];
    return 1;
}
```
Здесь вся логика держится на двух операциях:
heapify_up — восстановление свойства кучи после вставки.  
heapify_down — восстановление после удаления корня.
Физически дерево никогда не строится — оно только логически существует поверх массива.


---
В C++ STL куча напрямую не представлена как контейнер, но существует адаптер:
C++ Standard Library `std::priority_queue`.
Это именно **heap-адаптер**.
По умолчанию создаётся max-heap:
```C
#include <queue>
#include <iostream>

int main()
{
    std::priority_queue<int> q;

    q.push(10);
    q.push(30);
    q.push(20);

    std::cout << q.top() << std::endl; // 30

    q.pop();

    std::cout << q.top() << std::endl; // 20
}
```
Внутри используется `std::vector` + алгоритмы кучи.
Min-heap задаётся компаратором:
```C
#include <queue>
#include <vector>
#include <functional>

std::priority_queue<
    int,
    std::vector<int>,
    std::greater<int>
> q;
```
Отдельно важно: STL также предоставляет **алгоритмы работы с кучей** над произвольным массивом:
std::make_heap  
std::push_heap  
std::pop_heap  
std::sort_heap
Они работают с диапазонами итераторов и позволяют превратить любой `vector` в heap без адаптера.
Пример:
```C
#include <vector>
#include <algorithm>

std::vector<int> v = {3,1,5,2};

std::make_heap(v.begin(), v.end());
```
После этого `v[0]` — максимум.

Типовые применения кучи:
- приоритетные очереди  
- алгоритм Дейкстры  
- A*  
- heap sort  
- планировщики задач  
- event loop симуляций
Фактически куча — это структура для ситуаций, где постоянно требуется извлекать элемент с наивысшим приоритетом при динамическом добавлении данных.

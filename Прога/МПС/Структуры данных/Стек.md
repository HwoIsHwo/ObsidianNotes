Стек — это линейная структура данных с дисциплиной доступа **LIFO** (last in — first out). Доступ разрешён только к одному концу структуры, который обычно называют вершиной стека (_top_). Добавление элемента называется _push_, извлечение — _pop_. Просмотр без удаления — _top_ или _peek_.

Фактически стек — это ограниченный интерфейс над последовательным контейнером. Внутренняя реализация может быть любой: массив, связный список, даже дек.

Основные операции имеют константную сложность O(1), поскольку работа всегда ведётся только с вершиной.

---
Минимальная реализация стека на C обычно делается двумя способами: через массив или через односвязный список. Массив проще и быстрее, но требует фиксированного размера.
Простейший вариант на массиве:
```C
#include <stdio.h>
#include <stdlib.h>

#define STACK_CAPACITY 100

typedef struct
{
    int data[STACK_CAPACITY];
    int top;   // индекс вершины
} Stack;

// инициализация
void stack_init(Stack* s)
{
    s->top = -1;
}

// проверка пустоты
int stack_empty(Stack* s)
{
    return s->top == -1;
}

// добавление элемента
int stack_push(Stack* s, int value)
{
    if (s->top >= STACK_CAPACITY - 1)
        return 0; // переполнение

    s->data[++s->top] = value;
    return 1;
}

// извлечение элемента
int stack_pop(Stack* s, int* value)
{
    if (stack_empty(s))
        return 0;

    *value = s->data[s->top--];
    return 1;
}

// просмотр вершины
int stack_top(Stack* s, int* value)
{
    if (stack_empty(s))
        return 0;

    *value = s->data[s->top];
    return 1;
}

int main()
{
    Stack s;
    stack_init(&s);

    stack_push(&s, 10);
    stack_push(&s, 20);

    int v;
    stack_pop(&s, &v);
    printf("%d\n", v); // 20
}
```
Ключевой момент — индекс `top`.  
`-1` означает пустой стек. Push увеличивает индекс, pop уменьшает.

Альтернативный вариант — стек на связном списке. Он динамический и не имеет фиксированного размера:
```C
#include <stdio.h>
#include <stdlib.h>

typedef struct Node
{
    int value;
    struct Node* next;
} Node;

typedef struct
{
    Node* top;
} Stack;

void stack_init(Stack* s)
{
    s->top = NULL;
}

void stack_push(Stack* s, int value)
{
    Node* n = malloc(sizeof(Node));
    n->value = value;
    n->next = s->top;
    s->top = n;
}

int stack_pop(Stack* s, int* value)
{
    if (!s->top)
        return 0;

    Node* tmp = s->top;
    *value = tmp->value;

    s->top = tmp->next;
    free(tmp);
    return 1;
}
```
Здесь вершина — просто голова списка.


---
В C++ стек уже присутствует в STL как адаптер контейнера:  
C++ Standard Library `std::stack`.
Важно: `std::stack` — **не контейнер**, а адаптер. По умолчанию он использует `std::deque`, но можно подставить `std::vector` или `std::list`.
Пример использования:
```C
#include <stack>
#include <iostream>

int main()
{
    std::stack<int> s;

    s.push(10);
    s.push(20);
    s.push(30);

    std::cout << s.top() << std::endl; // 30

    s.pop();

    std::cout << s.top() << std::endl; // 20
}
```
Типичная особенность STL-стека — отсутствуют итераторы. Это сделано намеренно: стек должен ограничивать способ доступа к данным.

Можно явно задать базовый контейнер:
```C
#include <stack>
#include <vector>

std::stack<int, std::vector<int>> s;
```
Практически стек используется в следующих задачах:
• рекурсивные вызовы (call stack процессора)  
• парсинг выражений  
• обход графов и деревьев (DFS)  
• откат состояний (undo/redo)  
• вычисление выражений в обратной польской записи

Фактически любой алгоритм, где требуется возврат к последнему состоянию, естественно выражается через стек.

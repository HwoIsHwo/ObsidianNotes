Почти полное бинарное дерево (Complete Binary Tree, CBT) — это бинарное дерево, в котором все уровни, кроме возможно последнего, полностью заполнены, а узлы последнего уровня располагаются **строго слева направо без пропусков**.

Это ключевое свойство. В отличие от Full Binary Tree, здесь допускаются вершины с одним ребёнком, но только на последнем уровне и только справа отсутствующие.

Главная причина практической ценности CBT — возможность хранения **без указателей**, в линейном массиве. Структура дерева однозначно восстанавливается по индексу.

Если узел имеет индекс `i`, тогда:
```
left  = 2*i + 1
right = 2*i + 2
parent = (i - 1) / 2
```
Именно поэтому CBT используется в кучах (heap), priority queue, scheduler’ах и многих embedded-алгоритмах.


### Минимальная реализация CBT на C
На практике CBT почти всегда реализуется как динамический массив.
Ниже минимальный бинарный heap без балансировок — только структура CBT.
```C
#include <stdio.h>
#include <stdlib.h>

typedef struct
{
    int* data;
    size_t size;
    size_t capacity;
} CBT;

/* создание */
CBT* cbt_create(size_t capacity)
{
    CBT* t = malloc(sizeof(CBT));
    t->data = malloc(sizeof(int) * capacity);
    t->size = 0;
    t->capacity = capacity;
    return t;
}

/* добавление узла (структура CBT сохраняется автоматически) */
void cbt_push(CBT* t, int value)
{
    if (t->size >= t->capacity)
        return;

    t->data[t->size++] = value;
}

/* доступ к потомкам */
int left_child(CBT* t, size_t i)
{
    size_t idx = 2*i + 1;
    return (idx < t->size) ? t->data[idx] : -1;
}

int right_child(CBT* t, size_t i)
{
    size_t idx = 2*i + 2;
    return (idx < t->size) ? t->data[idx] : -1;
}

void print_tree(CBT* t)
{
    for (size_t i = 0; i < t->size; ++i)
        printf("%d ", t->data[i]);
    printf("\n");
}

void cbt_free(CBT* t)
{
    free(t->data);
    free(t);
}

int main()
{
    CBT* tree = cbt_create(16);

    for (int i = 1; i <= 7; ++i)
        cbt_push(tree, i);

    print_tree(tree);

    printf("children of 1: %d %d\n",
           left_child(tree,0),
           right_child(tree,0));

    cbt_free(tree);
}
```
Замечание: вставка всегда идёт в конец массива. Поэтому структура автоматически остаётся complete binary tree. Никакой явной логики дерева не требуется.


### CBT в STL
В STL **есть прямая эксплуатация CBT**, но не как отдельного контейнера.
Все heap-алгоритмы работают именно с complete binary tree, размещённым в массиве:
- `std::make_heap`
- `std::push_heap`
- `std::pop_heap`
- `std::sort_heap`
- `std::priority_queue`

Фактически STL-heap = CBT + heap property.
Пример:
```C++
#include <vector>
#include <algorithm>
#include <iostream>

int main()
{
    std::vector<int> v = {4, 1, 7, 3, 8, 5};

    std::make_heap(v.begin(), v.end());

    std::push_heap(v.begin(), v.end());

    std::pop_heap(v.begin(), v.end());
    v.pop_back();

    for (int x : v)
        std::cout << x << " ";
}
```
`std::vector` здесь физически хранит CBT. Алгоритмы просто поддерживают дополнительное условие кучи.

### `std::priority_queue`
Это уже готовый контейнер-адаптер:
```C++
#include <queue>
#include <iostream>

int main()
{
    std::priority_queue<int> pq;

    pq.push(10);
    pq.push(3);
    pq.push(20);

    while (!pq.empty())
    {
        std::cout << pq.top() << " ";
        pq.pop();
    }
}
```

Дерево — это иерархическая структура данных, состоящая из узлов, связанных отношением «родитель–потомок». В отличие от линейных структур (массив, список), каждый узел может иметь несколько потомков. Формально это связный ациклический граф с выделенным корнем.

Базовые свойства: есть корень, у каждого узла (кроме корня) один родитель, циклов нет. Глубина — расстояние от корня, высота — максимальная глубина поддерева. Важная характеристика — ветвление (сколько детей у узла) и баланс, так как от него зависит сложность операций.

Деревья применяются там, где данные имеют иерархию или когда требуется логарифмический доступ при упорядочении. Типичные случаи — файловые системы, AST компилятора, индексы баз данных, словари, множества. Если дерево сбалансировано, поиск, вставка и удаление работают за O(log n). В несбалансированном случае возможна деградация до O(n).

Минимальная абстрактная реализация на C без привязки к типу дерева может выглядеть так (узел с произвольным числом потомков через односвязный список детей):

```C
#include <stdio.h>
#include <stdlib.h>

typedef struct Node Node;

typedef struct Child {
    Node* node;
    struct Child* next;
} Child;

struct Node {
    int value;
    Child* children;
};

Node* create_node(int value) {
    Node* n = (Node*)malloc(sizeof(Node));
    n->value = value;
    n->children = NULL;
    return n;
}

void add_child(Node* parent, Node* child) {
    Child* c = (Child*)malloc(sizeof(Child));
    c->node = child;
    c->next = parent->children;
    parent->children = c;
}

void print_tree(Node* root, int depth) {
    for (int i = 0; i < depth; ++i)
        printf("  ");
    printf("%d\n", root->value);

    Child* c = root->children;
    while (c) {
        print_tree(c->node, depth + 1);
        c = c->next;
    }
}

void free_tree(Node* root) {
    Child* c = root->children;
    while (c) {
        Child* tmp = c;
        free_tree(c->node);
        c = c->next;
        free(tmp);
    }
    free(root);
}

int main() {
    Node* root = create_node(1);
    Node* n2 = create_node(2);
    Node* n3 = create_node(3);
    Node* n4 = create_node(4);

    add_child(root, n2);
    add_child(root, n3);
    add_child(n2, n4);

    print_tree(root, 0);
    free_tree(root);
    return 0;
}
```

Это обобщённое дерево произвольной степени. Память полностью управляется вручную. В реальной системе стоит продумать стратегию владения и, возможно, пул аллокаторов.

В C++ STL нет явного контейнера «tree», но многие ассоциативные контейнеры реализованы на основе сбалансированных бинарных деревьев (обычно красно-чёрных). Например, `std::set` и `std::map`.

Пример использования `std::set`:
```C++
#include <iostream>
#include <set>

int main() {
    std::set<int> s;

    s.insert(5);
    s.insert(2);
    s.insert(8);
    s.insert(5); // дубликат не добавится

    for (int v : s) {
        std::cout << v << " ";
    }
    std::cout << std::endl;

    auto it = s.find(2);
    if (it != s.end()) {
        s.erase(it);
    }

    return 0;
}
```
`std::set` хранит элементы упорядоченными и обеспечивает логарифмическую сложность операций. Структура дерева и балансировка скрыты внутри контейнера.

Если нужна именно иерархия произвольной формы (как в первом примере), в STL готового контейнера нет — обычно используют `std::vector<std::unique_ptr<Node>>` или `std::list` для хранения потомков и строят собственную структуру.

Ключевой момент: дерево выигрывает там, где требуется структурированная иерархия или логарифмический доступ к упорядоченным данным. Оно проигрывает массиву по локальности памяти и простоте, а односвязному списку — по простоте реализации. Выбор всегда определяется характером операций и требованиями к сложности.

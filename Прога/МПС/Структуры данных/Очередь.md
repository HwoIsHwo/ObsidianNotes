Очередь — это структура данных с дисциплиной FIFO (first in, first out). Элемент добавляется в конец, извлекается из начала. Семантически это модель «буфера», через который данные проходят в порядке поступления. Типовые применения — обработка задач, буферизация данных, межпоточное взаимодействие, BFS в графах.

С точки зрения реализации есть два базовых подхода:
1. на основе связанного списка;
2. на основе массива с кольцевой адресацией (circular buffer).

Для системного и embedded-кода чаще используют кольцевой буфер: он не требует динамического выделения памяти и имеет предсказуемое время выполнения.

Минимальная реализация очереди фиксированного размера на C через кольцевой буфер:
```C
#include <stdio.h>
#include <stdbool.h>

#define CAPACITY 5

typedef struct {
    int data[CAPACITY];
    int head;   // индекс чтения
    int tail;   // индекс записи
    int size;
} Queue;

void init_queue(Queue* q)
{
    q->head = 0;
    q->tail = 0;
    q->size = 0;
}

bool enqueue(Queue* q, int value)
{
    if (q->size == CAPACITY)
        return false; // очередь полна

    q->data[q->tail] = value;
    q->tail = (q->tail + 1) % CAPACITY;
    q->size++;
    return true;
}

bool dequeue(Queue* q, int* out)
{
    if (q->size == 0)
        return false; // очередь пуста

    *out = q->data[q->head];
    q->head = (q->head + 1) % CAPACITY;
    q->size--;
    return true;
}

int main()
{
    Queue q;
    init_queue(&q);

    enqueue(&q, 10);
    enqueue(&q, 20);
    enqueue(&q, 30);

    int value;
    while (dequeue(&q, &value))
        printf("%d ", value);

    return 0;
}
```

Здесь операции `enqueue` и `dequeue` работают за O(1), без сдвигов данных. Это важно: если реализовать очередь на «простом массиве» со сдвигом элементов при удалении, сложность станет O(n), что неприемлемо для интенсивных операций.

В C++ стандартная библиотека предоставляет адаптер контейнера — std::queue. Это не самостоятельная структура хранения, а обёртка над другим контейнером (по умолчанию `std::deque`).

Пример использования:
```C
#include <queue>
#include <iostream>

int main()
{
    std::queue<int> q;

    q.push(1);
    q.push(2);
    q.push(3);

    while (!q.empty())
    {
        std::cout << q.front() << " ";
        q.pop();
    }

    return 0;
}
```

`std::queue` предоставляет минимальный интерфейс: `push`, `pop`, `front`, `back`, `empty`, `size`. Нет итераторов и произвольного доступа — это сознательное ограничение, чтобы сохранить семантику FIFO.

Важно различать `std::queue` и `std::deque`. Первая — логический интерфейс очереди, вторая — конкретная структура хранения с доступом по индексу.

С инженерной точки зрения выбор реализации зависит от требований. Если нужен фиксированный буфер с предсказуемым временем выполнения (например, в ISR или RTOS) — кольцевая очередь без динамической памяти. Если нужна гибкость и автоматическое управление памятью — стандартный контейнер. В многопоточной среде сама по себе очередь не является потокобезопасной; требуется внешняя синхронизация или специализированная lock-free реализация.

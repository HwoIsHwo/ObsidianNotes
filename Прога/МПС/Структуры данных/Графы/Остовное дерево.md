Остовное дерево (_spanning tree_) — это подграф связного ненаправленного графа, который содержит **все вершины исходного графа**, но при этом является деревом. То есть выполняются три условия:
- граф остаётся связным  
- циклы отсутствуют  
- число рёбер равно V−1
Если исходный граф содержит `V` вершин, любое его остовное дерево всегда имеет ровно `V−1` ребро. Это фундаментальное свойство дерева.
Интуитивно остовное дерево — это минимальный набор связей, позволяющий добраться из любой вершины в любую другую без избыточных соединений.

---
Рассмотрим пример.
Исходный граф:
```
A --- B
| \   |
|  \  |
C --- D
```
Он содержит циклы. Один из возможных остовов:
```
A --- B
|
C --- D
```
Все вершины сохранены, но циклы удалены. Важно: у одного графа обычно существует **много** различных остовных деревьев.

---

Остов можно построить обычным обходом графа. Например, DFS или BFS автоматически формируют остовное дерево. Идея простая:
- мы идём по графу  
- каждое первое посещение вершины фиксируем ребром  
- повторные переходы игнорируем
Рёбра первого посещения и образуют spanning tree.
Минимальный пример построения остова через DFS (концептуально):
```C
void dfs(int v, int parent,
         int** adj, int* visited, int n)
{
    visited[v] = 1;

    for (int i = 0; i < n; ++i)
    {
        if (!adj[v][i])
            continue;

        if (!visited[i])
        {
            printf("tree edge: %d - %d\n", v, i);
            dfs(i, v, adj, visited, n);
        }
    }
}
```
Каждое напечатанное ребро принадлежит остовному дереву.

Особый случай — **минимальное остовное дерево** (_Minimum Spanning Tree, MST_). Если рёбра имеют веса, возникает задача:
- соединить все вершины  
- без циклов  
- с минимальной суммой весов
Это одна из ключевых задач теории графов и инженерных приложений. Основные алгоритмы:
- алгоритм Краскала  
- алгоритм Прима
Применения:
- проектирование сетей связи  
- электросети  
- дорожные системы  
- кластеризация данных  
- mesh-генерация

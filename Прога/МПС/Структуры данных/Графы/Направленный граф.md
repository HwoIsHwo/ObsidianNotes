Направленный граф (_directed graph_, digraph) — это граф, в котором каждое ребро имеет направление. Связь задаётся как упорядоченная пара вершин:
$$ u \rightarrow v$$
Это означает существование ребра **из** `u` **в** `v`, но обратное ребро автоматически не существует.
Формально:
G=(V,E), E⊆V×V
В отличие от ненаправленного графа, у вершины появляются две разные характеристики:
- входящая степень (_in-degree_) — сколько рёбер входит  
- исходящая степень (_out-degree_) — сколько рёбер выходит
Направленные графы используются значительно чаще ненаправленных, поскольку большинство реальных зависимостей асимметричны: вызовы функций, зависимости модулей, сетевые маршруты, конечные автоматы, граф задач.

Наиболее естественное представление — список смежности, где каждая вершина хранит **исходящие** рёбра.
Минимальная реализация направленного графа на C:
```C
#include <stdio.h>
#include <stdlib.h>

typedef struct Node
{
    int vertex;
    struct Node* next;
} Node;

typedef struct
{
    int vertices;
    Node** adj;   // списки исходящих рёбер
} Graph;

Node* create_node(int v)
{
    Node* n = malloc(sizeof(Node));
    n->vertex = v;
    n->next = NULL;
    return n;
}

Graph* graph_create(int vertices)
{
    Graph* g = malloc(sizeof(Graph));
    g->vertices = vertices;
    g->adj = calloc(vertices, sizeof(Node*));
    return g;
}

// ребро u -> v
void graph_add_edge(Graph* g, int u, int v)
{
    Node* n = create_node(v);
    n->next = g->adj[u];
    g->adj[u] = n;
}

void graph_print(Graph* g)
{
    for (int i = 0; i < g->vertices; ++i)
    {
        printf("%d ->", i);

        Node* cur = g->adj[i];
        while (cur)
        {
            printf(" %d", cur->vertex);
            cur = cur->next;
        }
        printf("\n");
    }
}

int main()
{
    Graph* g = graph_create(4);

    graph_add_edge(g, 0, 1);
    graph_add_edge(g, 0, 2);
    graph_add_edge(g, 2, 3);

    graph_print(g);
}
```
Ключевое отличие от ненаправленного графа — ребро добавляется только один раз. 
Если требуется двусторонняя связь, добавляются два направленных ребра.

В стандартной библиотеке C++ контейнера «граф» нет. Используется композиция STL-контейнеров.
Типичная реализация направленного графа:
```C++
#include <vector>

using Graph = std::vector<std::vector<int>>;

int main()
{
    Graph g(4);

    g[0].push_back(1); // 0 -> 1
    g[0].push_back(2); // 0 -> 2
    g[2].push_back(3); // 2 -> 3
}
```
`g[u]` содержит список вершин, достижимых из `u`.
Для взвешенного направленного графа:
```C++
#include <vector>
#include <utility>

using Graph =
    std::vector<std::vector<std::pair<int,int>>>;
// вершина + вес
```
Практически важные свойства направленных графов:
- наличие циклов  
- сильная связность  
- достижимость  
- топологический порядок (только для DAG)

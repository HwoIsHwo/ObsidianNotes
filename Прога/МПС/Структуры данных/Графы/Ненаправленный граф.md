Ненаправленный граф — это граф, в котором ребро соединяет две вершины симметрично. Связь между вершинами определяется неупорядоченной парой:
$$
u \leftrightarrow v
$$
Если существует ребро между `u` и `v`, то переход возможен в обе стороны. Формально:
(u,v)=(v,u)
Поэтому степень вершины определяется просто количеством инцидентных рёбер — разделения на входящие и исходящие нет. Такая модель естественно описывает физические соединения: дороги, электрические сети, топологию устройств, mesh-соединения и т.п.

Как и в случае любого графа, основные способы хранения — матрица смежности и список смежности. В реальных задачах почти всегда используется список смежности, поскольку память масштабируется как O(V + E). Главное отличие от направленного графа: **каждое ребро хранится дважды**.

Минимальная реализация ненаправленного графа на C.
```C++
#include <stdio.h>
#include <stdlib.h>

typedef struct Node
{
    int vertex;
    struct Node* next;
} Node;

typedef struct
{
    int vertices;
    Node** adj;   // список смежности
} Graph;

Node* create_node(int v)
{
    Node* n = malloc(sizeof(Node));
    n->vertex = v;
    n->next = NULL;
    return n;
}

Graph* graph_create(int vertices)
{
    Graph* g = malloc(sizeof(Graph));
    g->vertices = vertices;
    g->adj = calloc(vertices, sizeof(Node*));
    return g;
}

// ребро u <-> v
void graph_add_edge(Graph* g, int u, int v)
{
    Node* n1 = create_node(v);
    n1->next = g->adj[u];
    g->adj[u] = n1;

    Node* n2 = create_node(u);
    n2->next = g->adj[v];
    g->adj[v] = n2;
}

void graph_print(Graph* g)
{
    for (int i = 0; i < g->vertices; ++i)
    {
        printf("%d:", i);

        Node* cur = g->adj[i];
        while (cur)
        {
            printf(" %d", cur->vertex);
            cur = cur->next;
        }
        printf("\n");
    }
}

int main()
{
    Graph* g = graph_create(4);

    graph_add_edge(g, 0, 1);
    graph_add_edge(g, 1, 2);
    graph_add_edge(g, 2, 3);

    graph_print(g);
}
```
Здесь структура симметрична: добавление одного логического ребра создаёт две записи в списках смежности.

В стандартной библиотеке C++ отдельной структуры «граф» нет. Обычно граф строится поверх контейнеров STL.
Минимальное представление ненаправленного графа:
```C++
#include <vector>

using Graph = std::vector<std::vector<int>>;

int main()
{
    Graph g(4);

    auto add_edge = [&](int u, int v)
    {
        g[u].push_back(v);
        g[v].push_back(u);
    };

    add_edge(0, 1);
    add_edge(1, 2);
    add_edge(2, 3);
}
```
Для взвешенного графа:
```C++
#include <vector>
#include <utility>

using Graph =
    std::vector<std::vector<std::pair<int,int>>>;
```
где `pair<vertex, weight>`.

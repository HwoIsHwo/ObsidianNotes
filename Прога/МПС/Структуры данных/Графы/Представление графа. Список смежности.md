Список смежности — это способ представления графа, при котором для каждой вершины хранится набор вершин, непосредственно с ней соединённых. Идея следующая. Пусть граф содержит `V` вершин. Вместо хранения полной матрицы связей создаётся массив размера `V`, где каждый элемент указывает на список соседей соответствующей вершины.
Логически структура выглядит так:
```
0 → 1 → 4
1 → 0 → 2
2 → 1 → 3
3 → 2
4 → 0
```
Здесь строка `i` содержит все вершины, в которые существует ребро из вершины `i`.

Главная причина использования списка смежности — эффективность памяти. Если граф содержит мало рёбер (что типично для реальных задач), матрица смежности тратит память впустую:
- матрица: O(V²)  
- список смежности: O(V + E)
Для больших разреженных графов разница критическая. Например, при 1 млн вершин матрица практически невозможна, тогда как список смежности остаётся рабочим.

Минимальная реализация списка смежности на C. Здесь используется массив указателей на связные списки.
```C
#include <stdio.h>
#include <stdlib.h>

typedef struct Node
{
    int vertex;
    struct Node* next;
} Node;

typedef struct
{
    int vertices;
    Node** adj;   // массив списков смежности
} Graph;

Node* create_node(int v)
{
    Node* n = malloc(sizeof(Node));
    n->vertex = v;
    n->next = NULL;
    return n;
}

Graph* graph_create(int vertices)
{
    Graph* g = malloc(sizeof(Graph));
    g->vertices = vertices;

    // каждый элемент — начало списка соседей
    g->adj = calloc(vertices, sizeof(Node*));
    return g;
}

// ненаправленное ребро
void add_edge(Graph* g, int u, int v)
{
    Node* n1 = create_node(v);
    n1->next = g->adj[u];
    g->adj[u] = n1;

    Node* n2 = create_node(u);
    n2->next = g->adj[v];
    g->adj[v] = n2;
}

void print_graph(Graph* g)
{
    for (int i = 0; i < g->vertices; i++)
    {
        printf("%d:", i);

        Node* cur = g->adj[i];
        while (cur)
        {
            printf(" %d", cur->vertex);
            cur = cur->next;
        }
        printf("\n");
    }
}
```
Физически структура памяти выглядит как:
```
adj
 ├── [0] → Node → Node → NULL
 ├── [1] → Node → NULL
 ├── [2] → Node → Node → Node
```

В современном C++ связные списки вручную почти не используются. Список смежности естественно выражается через контейнеры STL:
```C++
#include <vector>

using Graph = std::vector<std::vector<int>>;

Graph g(5);

g[0].push_back(1);
g[1].push_back(2);
g[2].push_back(3);
```
`vector<vector<int>>` фактически и есть список смежности.
Для взвешенного графа:
```C++
#include <vector>
#include <utility>

using Graph =
    std::vector<std::vector<std::pair<int,int>>>;
```
где хранится `(сосед, вес)`.

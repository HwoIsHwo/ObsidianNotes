Граф — это нелинейная структура данных, состоящая из узлов или вершин и ребер. Ребра соединяют любые два узла в графе, а узлы также называются вершинами.
![[Pasted image 20260228090855.png]]
Этот граф имеет множество вершин V = {1,2,3,4,5} и множество ребер E = { (1,2),(1,3),(2,3),(2,4),(2,5),(3,5),(4,50 }.

В структурах данных существуют различные типы графов, каждый из которых подробно описан ниже.

1. Конечный граф
Граф G=(V, E) называется конечным графом, если количество вершин и ребер в графе ограничено.
![[Pasted image 20260228090955.png]]

2. Бесконечный граф
Граф G=(V, E) называется конечным графом, если число вершин и ребер в графе бесконечно.
![[Pasted image 20260228091016.png]]

3. Тривиальный граф
Граф G = (V, E) является тривиальным, если он содержит только одну вершину и не имеет рёбер.

4. Простой граф
Если каждая пара вершин в графе G=(V, E) имеет только одно ребро, то это простой граф. В результате, существует только одно ребро, соединяющее две вершины, что описывает взаимно однозначное взаимодействие между двумя элементами.
![[Pasted image 20260228091102.png]]

5. Мультиграф
Если между парой вершин в графе G = (V, E) имеется множество ребер, то такой граф называется мультиграфом. В мультиграфе нет петель.
![[Pasted image 20260228091128.png]]

6. Нулевой граф
Это переработанная версия тривиального графа. Если существует несколько вершин, но их не соединяют рёбра, то граф G = (V, E) является нулевым графом.

7. Полный граф
Если граф G = (V, E) является простым графом, то он является полным. Он состоит из ребер, соединенных n вершинами. Он также известен как полный граф, потому что степень каждой вершины должна быть равна n-1.

8. Псевдограф
Если граф G = (V, E) содержит петлю помимо других ребер, он является псевдографом.
![[Pasted image 20260228091318.png]]

9. Регулярный граф
Если граф G = (V, E) является простым графом с одинаковой степенью вершины, то он является регулярным графом. Следовательно, каждый целый граф является регулярным графом.
![[Pasted image 20260228091339.png]]

10. Взвешенный граф
Граф G = (V, E) называется помеченным или взвешенным графом, потому что каждое ребро имеет значение или вес, представляющий стоимость прохождения этого ребра.
![[Pasted image 20260228091403.png]]

11. Ориентированный граф
Ориентированный граф, также называемый диграфом, представляет собой набор узлов, соединенных ребрами, каждое из которых имеет определенное направление.
![[Pasted image 20260228091436.png]]

12. Неориентированный граф
Неориентированный граф состоит из множества вершин и соединяющих их связей. Порядок двух соединенных вершин не имеет значения и не имеет направления. Неориентированный граф можно построить с конечным числом вершин и ребер.
![[Pasted image 20260228091454.png]]

13. Связный граф
Если между одной вершиной структуры данных графа и любой другой вершиной существует путь, то граф является связным.

14. Несвязный граф
Когда между вершинами нет ребер, соединяющих их, нулевой граф называется несвязным графом.

15. Циклический граф
Если граф содержит хотя бы один цикл, он считается циклическим.
![[Pasted image 20260228091557.png]]

16. Ациклический граф
Если в графе нет циклов, он называется ациклическим графом.
![[Pasted image 20260228091615.png]]

17. Направленный ациклический граф
Он также известен как направленный ациклический граф (DAG) и представляет собой граф с направленными ребрами, но без циклов. Ребра в нем представлены упорядоченной парой вершин, поскольку он направляет вершины и хранит некоторые данные.
![[Pasted image 20260228091638.png]]

18. Подграф
Вершины и рёбра графа, являющиеся подмножествами другого графа, называются подграфом.
![[Pasted image 20260228091658.png]]

Терминология графов в структурах данных
Ниже приведена основная терминология графов в структурах данных:
- Ребро — это одна из двух основных единиц, используемых для формирования графов. Каждое ребро имеет два конца, которые являются вершинами, к которым оно прикреплено.
- Если две вершины являются конечными точками одного и того же ребра, они смежны.
- Исходящие ребра вершины — это направленные ребра, указывающие на начало координат.
- Входящие ребра вершины — это направленные ребра, указывающие на конечную точку вершины.
- Общее количество ребер, встречающихся у вершины в графе, называется ее степенью.
- Исходящая степень вершины в ориентированном графе — это общее количество исходящих ребер, тогда как входящая степень — это общее количество входящих ребер.
- Вершина с входящей степенью, равной нулю, называется исходящей вершиной, а вершина с исходящей степенью, равной нулю, — конечной вершиной.
- Изолированная вершина — это вершина с нулевой степенью, которая не является конечной точкой ребра.
- Путь — это набор чередующихся вершин и ребер, причем каждая вершина соединена ребром.
- Путь, который начинается и заканчивается в одной и той же вершине, называется циклом.
- Путь с уникальными вершинами называется простым путем.
- Граф, состоящий из каждой пары вершин x, y, является сильно связным, если он содержит направленный путь от x к y и направленный путь от y к x.
- Направленный граф является слабо связным, если все его направленные ребра заменены ненаправленными ребрами, в результате чего получается связный граф. Вершины слабосвязного графа имеют хотя бы одну исходящую или входящую степень.
- Дерево — это связный лес. Первичная форма дерева называется корневым деревом, которое является свободным деревом.
- Остевой подграф, который также является деревом, известен как остовное дерево.
- Связная компонента — это наиболее связный подграф несвязного графа.
- Мост, который является ребром удаления, разорвет граф.
- Лес — это граф без циклов.
Далее вы рассмотрите представление графа в этом руководстве по структурам данных.

Если граф взвешенный, вместо единиц и нулей можно записать вес ребра.

Алгоритм обхода графа
Процесс посещения или обновления каждой вершины в графе называется обходом графа. Последовательность посещения вершин используется для классификации таких обходов. Обход графа является подмножеством обхода дерева.
Существует два метода реализации алгоритма обхода графа:
- Поиск в ширину
- Поиск в глубину

## Поиск в ширину (BFS)
BFS — это метод поиска узла в структуре данных графа, который соответствует набору критериев. Он начинается с корня графа и исследует все узлы на текущем уровне глубины, прежде чем перейти к узлам на следующем уровне глубины.
Для отслеживания дочерних узлов, которые были обнаружены, но еще не проверены, требуется больше памяти, обычно это очередь.
Алгоритм поиска в ширину
- Шаг 1: Рассмотрите граф, по которому вы хотите перемещаться.
- Шаг 2: Выберите любую вершину в вашем графе, например, v1, от которой вы хотите пройти по графу.
- Шаг 3: Рассмотрите любые две структуры данных для обхода графа. 
	- Массив посещенных вершин (размер графа) 
	- Структура данных «Очередь»
- Шаг 4: Начиная с вершины, вы добавляете в массив посещенных вершин, а затем добавляете смежные с v1 вершины в структуру данных «Очередь».
- Шаг 5: Теперь, используя принцип FIFO (первым пришел — первым обслужен), необходимо удалить элемент из очереди, поместить его в массив посещенных вершин, а затем вернуться в очередь, чтобы добавить смежные вершины удаленного элемента.
- Шаг 6: Повторяйте шаг 5 до тех пор, пока очередь не станет полной и не останется ни одной вершины для посещения.
![[Pasted image 20260228093244.png]]

## Поиск в глубину (DFS)
DFS — это метод поиска узла в структуре данных графа, который соответствует набору критериев. Алгоритм поиска в глубину (DFS) обходит или исследует структуры данных, такие как деревья и графы. Алгоритм DFS начинается с корневого узла и исследует каждую ветвь настолько далеко, насколько это возможно, прежде чем вернуться назад. Для отслеживания дочерних узлов, которые были обнаружены, но еще не исследованы, требуется больше памяти, обычно стек.
Алгоритм поиска в глубину
- Шаг 1: Рассмотрите граф, по которому вы хотите перемещаться.
- Шаг 2: Выберите любую вершину в нашем графе, скажем, v1, с которой вы хотите начать обход графа.
- Шаг 3: Рассмотрите любые две структуры данных для обхода графа.
	- Посещенный массив (размер графа)
	- Структура данных «Стек»
- Шаг 4: Вставьте v1 в первый блок массива и поместите все смежные узлы или вершины вершины v1 в стек.
- Шаг 5: Теперь, используя принцип FIFO, извлеките верхний элемент и поместите его в посещенный массив, поместив в него все соседние узлы извлеченного элемента.
- Шаг 6: Если верхний элемент стека уже присутствует в массиве, удалите его, а не вставляйте в посещенный массив.
- Шаг 7: Повторяйте шаг 6, пока структура данных «Стек» не станет пустой.

Применение графов в структурах данных
Ниже приведены некоторые примеры применения графов в структурах данных:
- Графы используются в информатике для отображения потока вычислений.
- Пользователи Facebook называются вершинами, и если они друзья, то между ними существует ребро. Система рекомендаций друзей в Facebook основана на теории графов.
- Вы сталкиваетесь с графом распределения ресурсов в операционной системе, где каждый процесс и ресурс рассматриваются вертикально. Ребра проводятся от ресурсов к назначенным функциям или от запрашивающего процесса к желаемому ресурсу. Если это приводит к образованию цикла, возникнет тупиковая ситуация.
- Веб-страницы называются вершинами во Всемирной паутине. Предположим, существует ссылка со страницы A на страницу B, которая может представлять собой ребро. Это приложение является иллюстрацией ориентированного графа.
- Системы преобразования графов манипулируют графами в памяти с помощью правил. Графовые базы данных хранят и запрашивают данные, структурированные графами, безопасным для транзакций и постоянным образом.

Минимальная реализация графа на C (список смежности, ненаправленный граф).
```C
#include <stdio.h>
#include <stdlib.h>

typedef struct Node
{
    int vertex;
    struct Node* next;
} Node;

typedef struct
{
    int vertices;
    Node** adj;   // массив списков смежности
} Graph;

Node* create_node(int v)
{
    Node* n = malloc(sizeof(Node));
    n->vertex = v;
    n->next = NULL;
    return n;
}

Graph* graph_create(int vertices)
{
    Graph* g = malloc(sizeof(Graph));
    g->vertices = vertices;

    g->adj = calloc(vertices, sizeof(Node*));
    return g;
}

// добавление ребра u <-> v
void graph_add_edge(Graph* g, int u, int v)
{
    Node* n1 = create_node(v);
    n1->next = g->adj[u];
    g->adj[u] = n1;

    Node* n2 = create_node(u);
    n2->next = g->adj[v];
    g->adj[v] = n2;
}

void graph_print(Graph* g)
{
    for (int i = 0; i < g->vertices; ++i)
    {
        Node* cur = g->adj[i];

        printf("%d:", i);
        while (cur)
        {
            printf(" %d", cur->vertex);
            cur = cur->next;
        }
        printf("\n");
    }
}

int main()
{
    Graph* g = graph_create(4);

    graph_add_edge(g, 0, 1);
    graph_add_edge(g, 1, 2);
    graph_add_edge(g, 2, 3);

    graph_print(g);
}
```

В **C++ STL полноценного контейнера графа нет**.  
Стандартная библиотека сознательно ограничена базовыми структурами.
Обычно граф собирается из стандартных контейнеров:
```C++
#include <vector>

using Graph = std::vector<std::vector<int>>;

int main()
{
    Graph g(4);

    g[0].push_back(1);
    g[1].push_back(2);
    g[2].push_back(3);
}
```
Это фактически список смежности.
Для взвешенного графа:

```C++
#include <vector>

using Graph =

```

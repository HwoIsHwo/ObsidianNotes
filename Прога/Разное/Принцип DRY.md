**DRY (Don’t Repeat Yourself)** — принцип: Каждое знание в системе должно иметь единственный источник истины.
Речь не просто про копипаст кода. Речь про **дублирование логики, правил, констант, форматов, бизнес-смысла**.
Если одно и то же знание повторяется в нескольких местах — это потенциальная точка рассинхронизации.

## 1. Простейшее нарушение — копипаст логики
Плохо:
```
double calculate_discount(double price) {
    return price * 0.9;
}

double calculate_total(double price) {
    return price * 0.9 + 5;  // та же логика скидки
}
```

## 2. Дублирование условий
Плохо:
```
if (user.role == "admin") {
    // ...
}

...

if (user.role == "admin") {
    // ...
}
```

## 3. Дублирование формата данных
Плохо:
```
std::string format_user(const User& u) {
    return u.name + ":" + u.email;
}

std::cout << user.name << ":" << user.email;
```

## 4. Дублирование магических чисел
Плохо:
```
if (buffer.size() > 1024) { ... }
```
Лучше
```
constexpr std::size_t MAX_BUFFER = 1024;
```

## DRY vs KISS
- DRY уменьшает дублирование знаний
- KISS ограничивает избыточную абстракцию

Они могут конфликтовать.  
Иногда повтор в 2–3 строки проще, чем создание сложного универсального механизма.
